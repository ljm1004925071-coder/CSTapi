# CST 几何变异和结构增删改指南

## 目标

本指南用于在复杂天线工程中安全地增减、删除、替换、布尔操作或参数化 CST 几何结构。几何变异必须可回溯、可恢复、可复现。

## 修改前检查

在执行任何结构变异前，先记录：

- 工程路径和是否为副本。
- 组件列表、solid 名称、sheet 名称、材料、坐标单位。
- 参数表和与目标结构相关的参数。
- 端口、边界、监视器、mesh 设置。
- 需要保护的结构，例如馈电端口、参考地、连接器、封装、边界空气盒。

不要在不理解对象作用时直接删除。先隐藏、复制、重命名或在副本中测试。

## 命名规范

推荐命名：

- 新结构：`mut_M0003_slot_feed_1`、`mut_M0004_parasitic_arm_L`。
- 被替换结构备份：`bak_M0003_original_patch`。
- 临时布尔工具体：`tool_M0003_slot_cut`.
- 组件：`mutations/M0003` 或 `design/D0003`.

命名必须包含 mutation ID，方便从 CST 树反查到记录文件。

## 安全变异策略

- 新增结构：优先直接创建新对象，保持原结构不动。
- 删除结构：优先复制工程；必要时先把原对象重命名为 `bak_*`，不要直接永久删除。
- 开槽/挖空：创建 `tool_*` 工具体，再用 Boolean subtract；记录工具体尺寸和目标对象。
- 替换结构：保留原对象备份，创建新对象，确认端口/材料/边界仍正确。
- 批量修改：拆成多个 mutation，不要把多个物理机制混在一条 History 中。
- Rebuild 失败：记录失败 History caption，回滚到父设计，不要继续叠加修改。

## History/VBA 操作记录

每次 `add_to_history()` 的 caption 建议格式：

```text
M0003 D0002->D0003 add slot near feed
```

每个操作记录：

```json
{
  "mutation_id": "M0003",
  "history_caption": "M0003 D0002->D0003 add slot near feed",
  "operation": "boolean_subtract",
  "target_objects": ["metal:patch"],
  "tool_objects": ["tool_M0003_slot_cut"],
  "created_objects": ["mut_M0003_slot_feed_1"],
  "deleted_or_hidden_objects": [],
  "parameters": {"slot_l": "8 mm", "slot_w": "0.6 mm"},
  "rebuild_status": "completed"
}
```

## 常见几何动作

### 加槽

用途：拉长电流路径、引入新谐振、调匹配、改变极化。

注意：

- 槽靠近强电流区域通常更有效，但也更可能降低效率。
- 槽宽过窄会引入制造风险。
- 槽的位置和方向要和目标电流路径相关，不要随机切。

### 加寄生结构

用途：展宽带宽、增加耦合谐振、改善方向图或增益。

注意：

- 记录寄生结构和主辐射体之间的间距。
- 先用参数控制距离、长度和宽度。
- 关注效率和方向图副瓣变化。

### 改馈电结构

用途：改善输入匹配、阻抗变换、带宽。

注意：

- 修改馈电前记录端口定义和参考阻抗。
- 改馈线宽度、馈点位置、间隙、匹配枝节时，一次只改一个主要因素。
- 修改后检查端口是否仍连接到正确导体。

### 改地板

用途：调谐、改变方向图、改善隔离、降低尺寸。

注意：

- 地板开槽可能带来背辐射和效率变化。
- 小型化通常会牺牲带宽或效率。
- 阵列/多端口结构中，地板变异会影响隔离。

### 加短路或过孔

用途：小型化、引入新模式、改变极化或匹配。

注意：

- 明确短路连接的两个对象或层。
- 检查金属连续性和网格质量。
- 记录过孔半径、位置和数量。

## 回滚规则

- 如果 rebuild 失败，保留失败记录，回到 `parent_design_id`。
- 如果仿真失败但 rebuild 成功，可以保留设计作为 `failed_simulation`，不要进入下一轮优化。
- 如果指标明显变差，标记为 `rejected`，但保留数据，因为负样本对代理模型有用。
- 如果修改破坏端口或边界，优先回滚，而不是在错误结构上继续修补。

## 输出要求

几何修改任务完成后报告：

- 修改对象和新对象名称。
- 使用的 History caption。
- 参数和单位。
- Rebuild 状态。
- 是否保存副本。
- 结构版本记录路径。
- 回滚或失败处理结果。
