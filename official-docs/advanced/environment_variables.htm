Environment Variables

CST Studio Suite features the following environment variables:

 

CST_WORK_DIR=<directory>

Sets a designated working directory which is used for temporary files during startup or for untitled projects. In a cluster environment it might make sense to set this variable to be able to clean up all temporary files afterwards. If neither CST_WORK_DIR variable nor DefaultWorkingDir in HKEY_CURRENT_USER\Software\CST AG\CST STUDIO SUITE\Usersettings under Windows or $HOME/.config/CST AG/CST Studio Suite.conf under Linux is set, CST Studio Suite deletes the temporary working directory at the end of the batch job.

CST_WAIT_FOR_LICENSE=<arg>

Valid values for <arg> are 0 or 1. By default, a batch job is canceled if the license server is not available or all license are currently in use. Set this environment variable to wait for the license server become available again.

CST_FORCE_SOFTWARE_RENDERER=<arg>

Linux only: Valid values for <arg> are 0 or 1. The CST_FORCE_SOFTWARE_RENDERER acts as a main switch for either disabling (=0) or enabling (=1) the software renderer. By default, CST Studio Suite tries to automatically detect the best scenario based on a variety of factors, like remote (SSH, or VNC-like) connections, headless displays, batchmode or interactive physical display. CST Studio Suite uses GPU accelerated 3D rendering by default when started interactively, However, some GPU drivers are unable to handle OpenGL commands properly. If you experience issues with 3D rendering on your system you may  switch off GPU accelerated rendering by setting this environment variable.  

CST_VERBOSE_FRONTEND=0

Linux only: Redirects all output to temporary file. By default all output is written to the console.
