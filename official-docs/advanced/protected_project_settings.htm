Protected Project Settings

In order to automate the creation of projected projects, one can define the protected project settings in a JSON file. This JSON file can then be used when creating protected projects either using the command line option or using scripts.

 

The content of the JSON file can be defined as shown here:

 

{

 "destination_folder"  : "E:\\Protected",

 "placeholders"        : "component1:solid2",

 "unprotect_shapes"    : "component1:solid1_1",

 "unprotect_wires"     : ["Folder1:wire2", "wire1"],

 "unprotect_ports"     : true,

 "unprotect_materials" : false

}

 

It is not mandatory to specify all the below mentioned sections. You could specify one or more sections in the file depending on your workflow.

 

destination_folder:

Specify a string.

The string defines where the protected projects should be saved.

The protected projects are normally saved at the same location as that of the original project.

The project name is automatically suffixed with '_protected'.

 

placeholders:

Specify a string or an array of strings.

The string can be a complete name of a solid or a component which you want to use as a placeholder shape.

The protected projects normally defines an automatic placeholder by performing a boolean unite operation of the bonding boxes of all the protected shapes and wires.

 

unprotect_shapes:

Specify a string, an array of strings, true or false.

The string can be a complete name of a solid or a component which you do not want to protect.

If you specify true, no shapes will be protected. If you specify false, all shapes will be protected.

The protected projects normally protects all shapes.

 

unprotect_wires:

Specify a string, an array of strings, true or false.

The string can be a complete name of a wire or a folder which you do not want to protect.

If you specify true, no wires will be protected. If you specify false, all wires will be protected.

The protected projects normally protects all wires.

 

unprotect_materials:

Specify a string, an array of strings, true or false.

The string can be a complete name of a material or a folder which you do not want to protect.

If you specify true, no materials will be protected. If you specify false, all materials will be protected.

The protected projects normally protects all materials except default materials i.e.  PEC, Vacuum and Air.

 

unprotect_ports:

Specify a string, an array of strings, true or false.

The string can be a complete name of a port or a folder which you do not want to protect.

If you specify true, no ports will be protected. If you specify false, all ports will be protected.

The protected projects normally protects all ports.

 

 

Example:

Protect only materials Protect only shapes and materials

{

 "unprotect_shapes"    : true,

 "unprotect_wires"     : true,

 "unprotect_ports"     : true,

 "unprotect_materials" : false

}

{

 "unprotect_shapes"    : false,

 "unprotect_wires"     : true,

 "unprotect_ports"     : true,

 "unprotect_materials" : false

}
