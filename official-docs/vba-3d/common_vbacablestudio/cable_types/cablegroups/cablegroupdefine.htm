CableGroupDefine

Create or modify a cable group. For the inner cable definition please refer to the SingleWireDefine, RibbonCableDefine, TwistedCableDefine and CoaxCableDefine commands.

For naming conventions and the idea behind hierarchical elements like screens please refer to Hierarchical structure definition.

Methods

Group

OutputName(string objType, string cmdId, string value)

Sets the name of the wire that will be edited or created in ’value’.

Parameter 'objType' is "CSCableGroup".

Parameter 'cmdId' is "Name".

AddStringArgument(string hierarchical, bool value)

perform hierarchical layout on autobundle "Groupname$hierarchical"

  with value true when sub cables are to be automatically bundled, false if the layout of sub cables shall be maintained.

AddArgument(string posRot, double value)

posRot values may be "Groupname$cablename$x", "Groupname$cablename$y", "Groupname$<string>$rotation"

AddStringArgument(string localUnitsKey, string value)

Key localUnitsKey named "LocalUnits" and value one of "m, cm, mm, um, nm, ft, in, mil" specifies the local unit the cable is defined in. All length, diameter, area, wrap thickness and so on values will be considered to be in this given unit.

 

Content of Group

AddStringArgument(string contentType, string elementName)

contentType may be one of "Groupname$<enum{ Screen, ShieldBreak, DrainWire, Insulator, Insulator inside}>"

elementName "Group$contentName", e.g. "MyBraidedScreen"

AddStringArgument(string indexName, integer idx)

indexName: "Groupname$elementName$Index" defines an index relative to other elements.

idx: relative position with regard to other elements on the same hierarchy levels. E.g. a stack of dielectrics, screens are sorted by this index value in ascending order.

 

Auto layout

This option allows to insert sub cables without the need to specify their exact positions. In many cases this is sufficient.

AddStringArgument(string param, string value)

param "GroupName$autolayout"

value "True" or "False"

AddArgument(string param, bool value)

param "GroupName$DiscontScreen$connect", value determines connection to ideal reference conductor (true/false)

"Group$drainName$x", "Group$drainName$y", value for the positioning of the drain wire

Cable group extended twist settings

AddArgument(string param, string value)

param "GroupName$Turn" one of "Left", "Right"

parem "GroupName$Lay_type" one of "Lay" or "Twist per meter"

param "GroupName$TwistType", "Twist" or "None"

param "GroupName$TwistCenter", one of "Auto", "User", "Cable"

param "GroupName$TwistCenterCable", the value contains the ID of the cable to be used as center in case of TwistCenter "Cable".

 

AddArgument(string param, double value)

param "GroupName$TwistCenterX", "Group$TwistCenterY", value determines the relative coordinates of the twist center for TwistCenter "User".

param "GroupName$Lay_value", value determines the numerical value to use in combination with "Lay" or "Twist per meter".
