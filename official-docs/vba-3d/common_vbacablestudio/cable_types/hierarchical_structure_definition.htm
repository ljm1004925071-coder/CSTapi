Hierarchical structure definition

For more complex cable and bundle definitions the user can define e.g. a cable in a bundle within a bundle.

 

     

 

The CST Cable Studio Graphical User Interface (GUI) is based on this principle and therefore the VBA interface provides the means to create corresponding structures like screens, sub-bundles etc. in a well defined and reproducible way..

 

In the example above the full naming of the sub cable is as follows:

TopGroup$Group_1$LIFY1

TopGroup$Group_1$LIFY2

TopGroup$Group_2$LIFY3

TopGroup$Group_2$LIFY4

 

The dollar ($) sign is used to denote the hierarchy between elements. The number of '$' signs in a command is not limited, the user can define arbitrary sized element structures that way.

 

This results in a cable definition (in this example for the single wire cable LIFY_1 mentioned above) that looks like this:

 

.AddStringArgument "LocalUnits", "mil"

Specify a the unit the cross section values are specified in. Can be any one of [m, cm, mm, um, nm, ft, in, mil]

.AddStringArgument "TopGroup$Group1$SingleWireCable", "TopGroup$Group1$LIFY_1"

Specify a member of the parent element (TopGroup$Group1) of type SingleWireCable with a given name.

.AddStringArgument "TopGroup$Group1$LIFY_1$conductor$shape", "Circle composed of strands"

Declares the shape type (stranded wire) of the conductor

.AddStringArgument "TopGroup$Group1$LIFY_1$conductor$strand_definedby", "Diameter"

Declares the way the geometry is defined (by diameter)

.AddArgument "TopGroup$Group1$LIFY_1$conductor$strand_diameter", "0.05"

States the actual value for the diameter

.AddArgument "TopGroup$Group1$LIFY_1$conductor$strand_numberofstrands", "256.0"

Specifies other parameter 'strand_numberofstrands'

.AddStringArgument "TopGroup$Group1$LIFY_1$insulator$shape", "Circle"

Declares the shape type of the insulator of the cable (circular)

.AddStringArgument "TopGroup$Group1$LIFY_1$insulator$material", "PVC"

Declares the material ID of the insulator

.AddStringArgument "TopGroup$Group1$LIFY_1$insulator$circle_definedby", "Diameter"

Declares the way the geometry is defined (by diameter)

.AddArgument "TopGroup$Group1$LIFY_1$insulator$circle_diameter", "2.0"

States the actual value for the diameter

.AddArgument "TopGroup$Group1$LIFY_1$x", "-1.0"

Defines the x-position inside the outer element

.AddArgument "TopGroup$Group1$LIFY_1$y", "-0.5"

Defines the y-position inside the outer element

.AddArgument "TopGroup$Group1$LIFY_1$rotation", "0.0"

Defines the rotation of the wire
