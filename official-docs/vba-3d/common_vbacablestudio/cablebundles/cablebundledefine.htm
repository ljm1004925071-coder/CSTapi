CableBundleDefine

Defines all elements in a cable bundle.

For naming conventions and the idea behind hierarchical segment elements like cables, bundles and sub-cables please refer to Hierarchical structure definition.

  • Methods

  • Keywords

  • Example

Methods

Reset

Resets all internal settings for this command to their default values.

OutputName(string objType, string cmdId, string name)

Sets the ’name’ of the cable bundle that will be edited or created.

objType ’CSBundle’, cmdID ’Name’ defines the name for this cable bundle.

AddInputName(string objType, string elementId, string nodeId)

objType ”CSNode”: add value ’nodeId’ to trace ”CSTrace$Trace_<int>”

objType ’CSBundle’, values are CSSingleWire, CSTwistedCable, CSRibbonCable, CSCoaxCable, CSCableGroup.

elementId is shaped in the form <bundleName>$<instanceName>

nodeId is the name the new cable will have in the defined cable bundle.

objType ’CSNode’ specifes a trace definition

elementId defines the hierarchical name of the trace ’<bundleName>$<traceId>’

nodeId specifies a name of an existing node

AddStringArgument(string elementId, bool value)

Determines if the cable mesher is allowed to randomly bundle the current bundle.

elementId "<Bundle_name>$RandomBundle", value "<boolean>"

Execute(string command)

Create or modify a cable bundle, command ”CableBundleDefine”

Keywords

CSBundle : object type (bundle)

CSNode: object type (node)

CSTrace: object type (trace)

RandomBundle: parameter setting random bundling

CableInstance: parameter setting a cable instance name

Bundle: parameter setting a bundle name

Cable instance

SegmentElement, RouteElement: parameter setting a cable instance type

x: x coordinate of the cable instance

y: y coordinate of the cable instance

rotation: rotation coordinate of the cable instance

CableType: parameter setting cable type of cable instance

CableTypeId: parameter setting id of existing cable corresponding to the cable instance

Bundle

SegmentBundle, RouteBundle: parameter setting a bundle type

x: x coordinate of the cable instance

y: y coordinate of the cable instance

Insulator: parameter setting insulator name

Index: parameter setting insulator index

skip_traces_def: If set to true the traces definition is skipped during execution of VBA command and only the bundle definition is considered.

Doesn’t create a new route (cable bundle) since to create a cable bundle traces definition is obligatory. Therefore it can only be applied to an existing cable bundle.

 

Example

With CableStudio

    .Reset

    .OutputName "CSBundle", "Name", "R_1"

    .AddInputName "CSNode", "CSTrace$Trace_1", "N3"

    .AddInputName "CSNode", "CSTrace$Trace_1", "N4"

    .AddStringArgument "R_1$RandomBundle", "true"

    .AddStringArgument "R_1$CableInstance", "SW_8"

    .AddStringArgument "R_1$SW_8", "RouteElement"

    .AddArgument "R_1$SW_8$x", "0.0"

    .AddArgument "R_1$SW_8$y", "0.0"

    .AddArgument "R_1$SW_8$rotation", "0.0"

    .AddStringArgument "R_1$SW_8$CableType", "CSSingleWire"

    .AddInputName "CSSingleWire", "R_1$SW_8$CableTypeId", "Single_Wire"

    .AddStringArgument "R_1$SW_8$Index", "0"

    .AddStringArgument "R_1$Bundle", "R_1_Bundle_1_1"

    .AddStringArgument "R_1$R_1_Bundle_1_1", "RouteBundle"

    .AddArgument "R_1$R_1_Bundle_1_1$x", "0.0"

    .AddArgument "R_1$R_1_Bundle_1_1$y", "0.0"

    .AddArgument "R_1$R_1_Bundle_1_1$rotation", "0.0"

    .AddStringArgument "R_1$R_1_Bundle_1_1$RandomBundle", "true"

    .AddStringArgument "R_1$R_1_Bundle_1_1$Insulator", "R_1$R_1_Bundle_1_1$Insulator"

    .AddStringArgument "R_1$R_1_Bundle_1_1$Insulator$Index", "0"

    .AddArgument "R_1$R_1_Bundle_1_1$Insulator", "0.0"

    .AddStringArgument "R_1$R_1_Bundle_1_1$Insulator$shape", "Wrap"

    .AddStringArgument "R_1$R_1_Bundle_1_1$Insulator$material", "PE"

    .AddArgument "R_1$R_1_Bundle_1_1$Insulator$wrap_thickness", "0.5"

    .AddStringArgument "R_1$R_1_Bundle_1_1$CableInstance", "SW_9"

    .AddStringArgument "R_1$R_1_Bundle_1_1$SW_9", "RouteElement"

    .AddArgument "R_1$R_1_Bundle_1_1$SW_9$x", "0.0"

    .AddArgument "R_1$R_1_Bundle_1_1$SW_9$y", "0.0"

    .AddArgument "R_1$R_1_Bundle_1_1$SW_9$rotation", "0.0"

    .AddStringArgument "R_1$R_1_Bundle_1_1$SW_9$CableType", "CSSingleWire"

    .AddInputName "CSSingleWire", "R_1$R_1_Bundle_1_1$SW_9$CableTypeId", "Single_Wire"

    .AddStringArgument "R_1$R_1_Bundle_1_1$SW_9$Index", "0"

    .AddStringArgument "R_1$R_1_Bundle_1_1$Index", "1"

    .Execute "CableBundleDefine"

End With
