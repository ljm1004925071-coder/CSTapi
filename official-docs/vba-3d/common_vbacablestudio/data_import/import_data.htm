Import Data

The command imports external data for geometry and cable information.

This includes:

NASTRAN CBAR (nodes & segments only)

Legacy projects from SimLab CableMod

KBL exchange data (geometry and optional cable data).

Methods

Reset

Resets all internal settings for this command to their default values.

AddStringArgument(string propertyId, string value)

Parameter 'propertyId' with value 'Type' defines the file type to import:

    value 'Nastran': Nastran CBAR data exported from e.g. Catia

    value 'KBL': Cable exchange data (see Harness Description List)

    value 'SMLB': Legacy file format of SimLab CableMod (slh & zlh format)

Parameter 'propertyId' with value 'FileName' defines the full path to the file to import in 'value'.

Parameter 'propertyId' with value 'CurveType' with 'value' to the following values will activate the alternative interpretation of spline curves.
   Legacy: Curve data is interpreted as simple Bezier splines
   Vobes: Curve data is interpreted as UNRBS (uniform non rational B-Splines)
   Catia: Curve data is interpreted as NURBS (non uniform rational B-Splines).

Parameter 'propertyId' with value 'MapFileName' defines the full path to the mapping file to use during import in 'value'. This value is optional.
This mapping file is used for the KBL import to provide the possibility to map company internal names to CST cable library names. The syntax of the file is described in more detail here.

Execute (string command)

Imports the given data into CST Cable Studio.

Interpretation of curves

Historically a curve in KBL is defined by using B-Splines. In recent years some vendors (namely the Vobes toolset of the VW group) extended that definition to UNRBS (uniform non rational B-Splines) which result in different curves from the point data contained in a KBL.

Data from Catia system can also be in NURBS format (non uniform rational B-Splines).

With the "CurveType" option the user can switch to the newer variants to match the curve data to the creating system.

KBL default cable name substitution

The KBL format has some deficiencies regarding the precise definition in the cross section information for more complex cables besides single wires. It is e.g. not possible to define the precise geometry for screened cables, the diameter, thickness and screen type parameters can not be specified in KBL.

To overcome this limitation it is possible to specify the name of an existing CST Cable Studio cable library element which will then be used instead.

The corresponding fields in a KBL harness definition file are Part_number and Wire_type.

  • Part_number: the ID of a specific cable type like "RG58" or "20 AWG".

  • Wire_type: the basic type of the cable being defined.
    CST Cable Studio recognises the following "Wire_type" keywords for the basic cable types:
    Singe wires: all cables with one core
    Twisted cables: "twist" or "drill"
    Ribbon cables: "flat" or "flach"
    Coaxial cables: "coax", "koax", "screen" or "schirm"
    Cable groups: all other multi-core wires

In case that there is a different naming scheme in place the users can either rename the elements in the CST Cable Studio cable library or use a separate mapping file in the import dialog to describe the name mapping

 

Example for a KBL cable cross section definition:

 

<General_wire id="GW_5">

    <Company_name>CST AG</Company_name>

    <Version>1.0</Version>

    <Abbreviation>/NULL</Abbreviation>

    <Description>Simple single wire cable</Description>

    <Part_number>LIFY_1qmm00</Part_number>

    <Wire_type>single</Wire_type>

    <Cross_section_area id="NumVal_10">

      <Value_component>1.25</Value_component>

      <Unit_component>Unit_1</Unit_component>

    </Cross_section_area>

    <Outside_diameter id="NumVal_11">

      <Value_component>3.26</Value_component>

      <Unit_component>Unit_1</Unit_component>

    </Outside_diameter>

  </General_wire>

KBL mapping file

This optional mapping file can simplify the import of KBL files by delegating the mapping of company specific naming schemes and the CST Cable Studio cable library to a separate file that can be managed more easily without the need to edit and modify the internal cable library in CST Cable Studio.

 

Syntax:

<?xml version="1.0" encoding="UTF-8"?>

<Mappings>

    <Mapping importID="first screened cable" cableLibID="RG58"/>

    <Mapping importID="second screened cable" cableLibID="RG174"/>

    <Mapping importID="third screened cable" cableLibID="RG316"/>

</Mappings>

 

Mapping parameter "importID" defines the name that may occur in the imported KBL file in the Part_number field of a "General_wire" element.

Mapping parameter "cableLibID" defines the name of an existing element in the CST Cable Studio cable library. All libraries will be searched and if a corresponding element is found its cross section information will be used during the import process to provide accurate cross section information.

Examples

With CableStudio

    .Reset

    .AddStringArgument "Type", "KBL"

    .AddStringArgument "MapFileName", "D:\Import\NameMapping.xml"

    .AddStringArgument "FileName", "D:\Import\29525-2014-0004_4525320-a-v23.kbl"

    .Execute "CableImport"

End With

 

With CableStudio

    .AddStringArgument "Type", "Nastran"
