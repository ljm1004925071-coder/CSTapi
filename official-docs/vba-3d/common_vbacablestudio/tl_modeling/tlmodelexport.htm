TLModelExport

This command family defines the export setup for external models to a file.

  • Methods

  • Example

Methods

Reset

Resets all internal settings for this command to their default values.

AddStringArgument (string name, string value)

To set up the parameters for the TLExportSettings

Parameter 'name' may be one of the following values corresponding to the options in the 2DTL dialog:

  ExportFile: 'value' contains a valid file path including the file name.

  ExportModelName: 'value' contains the name the exported model should have.

  ExportSimulator: 'value' may be one of the following external simulator names.

    "Spice3"

    "Matrix" (only available in case LF Reduction is on)

 

To perform the TLStartExport

Parameter 'name' may be one of the following values:

  "RefreshSelection" If true refresh the selected nets. Default is false, no refresh takes place.

  "Asynchronous" if true the model is created non-sequentially. Default is true.

For part of a VBA workflow it is recommended to use Asynchronous=false to avoid any race conditions so the model is present when the next command is executed.

 

Execute (string command)

Defines the input for the export model command or starts the export.

The parameter 'command' is either

"TLExportSettings" along with the parameters above

"TLStartExport " with the following parameters to trigger the export:

Examples

With CableStudio

    .Reset

    .AddStringArgument "ExportFile", "d:\export.lib"

    .AddStringArgument "ExportModelName", "X"

    .AddStringArgument "ExportSimulator", "Spice3"

    .Execute "TLExportSettings"

End With

 

With CableStudio

    .AddStringArgument "RefreshSelection", "false"

    .AddStringArgument "Asynchronous", "true"

    .Execute "TLStartExport"

End With
