DomainDecomposition Object

This object is used to manage the Domain Decomposition.

  • Methods

  • Default Settings

  • Example

Methods

Reset

Resets all internal settings to their default values.

 

Create

Creates a Domain Decomposition.

 

MinimumNumberOfDomains ( int ndomains )

Specify the minimum number of domains.

 

MaximumSizeInFreeWavelength( double nsize )

This factor times the wavelength in vacuum defines a maximum size for the domains in the base decomposition.

 

FactorTimesCADcomplexity( double nfactor )

Specify the factor for the CAD complexity. This setting only influences the minimum number of domains if the CAD model has many details.

 

EnclosePortsByDomains( bool bflag,  int nports )

Specify the ports enclosed by domains. Besides "automatic", the keywords "all", "discrete", and "waveguide" are supported. Lists of ports can be given by their numbers, separated with semi-colon, and ranges of port numbers separated by minus, for instance "1;2;42-101".

 

EncloseComponents( bool bflag,  int ncomponents )

Specify the components to enclose by domains. A part of the name of components is specified here, for instance "Element". Each component which has a name containing "Element" is then enclosed in a domain box with higher priority than the base decomposition.

 

StructuralSymmetry( string swhat, bool bflag )

If symmetries are defined, only a part of the structure is subject to the domain decomposition, namely the upper half space in the coordinate direction. In addition to those electric and magnetic symmetries, one structural symmetry can be defined.

 

UseAutomatic( bool bflag )

Activate automatic domain decomposition (default), or enable the manual decomposition.

 

StartSplit

Start the split to generate a manual decomposition, to refine the base decomposition.

 

Number( int nnumber )

Set the domain number for the manual split.

 

AddSplit( enum{"x", "y", "z"} direction,  string splits )

Add splits for the previously selected domain. &splits& is a space separated list of increasing values between zero and one, which indicate where to split the domain relative to its extent.

 

EndSplit

End the definition of splits for the manual refinement of a domain.

 

UseAutomaticRepetitions( bool bflag )

If activated, the grouping of all domains will be automatically performed at the solver start.

 

AddIdentification( string srefdomain, string sdepdomain )

Add identification between reference domain and dependent domain.

 

SetIdentifyByPoint( bool bpicked,  double x,  double y,  double z )

Identify repetitions semi-automatically, for those domains which contain the point coordinates.

 

SetJoinByPoint( bool bpicked,  double x,  double y,  double z )

Join domains which share the given point.

 

ProblemClass( string stype )

Set the problem type.

 

UnverifiedRepetitionsAllowed( bool ballowed )

Allow unverified repetitions.

 

AnalyzeMesherFeedbackAndJoinDomains

If the meshing has generated feedback information, this method may generate a history block with the "SetJoinByPoint" command, to enclose reported problems in larger domains.

 

FindAllRepetitionsAutomatically

Find all repetitiions.

Default Settings

MinimumNumberOfDomains (4)

MaximumSizeInFreeWavelength (1.0)

FactorTimesCADcomplexity (0.001)

UseAutomatic (True)

ProblemClass ("Automatic")

EncloseComponents (True, "Element")

UseAutomaticRepetitions (False)

Example

With DomainDecomposition

    .Reset

    .MinimumNumberOfDomains "5"

    .FactorTimesMPInodes "0"

    .MaximumSizeInFreeWavelength "1.0"

    .FactorTimesCADcomplexity "0.001"

    .EnclosePortsByDomains "True", "Automatic"

    .EncloseComponents "False", "Element"

    .StructuralSymmetry "x", "False"

    .StructuralSymmetry "y", "False"

    .StructuralSymmetry "z", "False"

    .ProblemClass "Automatic"

    .UseAutomatic "True"

    .Create

End With
