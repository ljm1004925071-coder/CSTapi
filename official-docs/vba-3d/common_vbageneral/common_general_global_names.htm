Global Names

Global Names are primary used in those VBA Commands that deal with different kind of objects. They are needed to be able to deal with those different kind of objects at the same time - with one VBA call. For some VBA commands, those names are mandatory, some more VBA commands support them, but also "local" names can be used.

 

Global Names are assembled by the "local name" of an object (See also: Solid Names, Name Data Type). In addition, a type specifier is prepended and a "$" sign is used to concatenate both parts.

Most of the VBA commands that support global names only offer a subset of types to work upon.

 

Most commonly used type specifiers:

Type

Type Specifier

Examples

Anchor points

anchorpoint

anchorpoint$subfolder:anchor1

Coils

coil

coil$coil1

Field Sources

fieldsource

fieldsource$fs1

Groups

group

group$meshgroup1

Lumped Elements

lumpeditem

lumpeditem$Folder1:element1

Imported Mesh Shape

meshshape

meshshape$Folder1/subfolder:shape1

Ports (Waveguide Ports as well as Discrete Ports)

port

port$port3

Shapes ("Solids")

solid

solid$component1/subcomponent_1:solid1

Voxel Data

voxeldata

voxeldata$Laura

Wires

wireitem

wireitem$wire1, wireitem$Folder1/subfolder:nice_wire
