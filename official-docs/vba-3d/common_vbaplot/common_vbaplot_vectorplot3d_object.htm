VectorPlot3D Object

This object allows to access different vector plot settings of the current 3D plot. To choose what vector field to plot, the SelectTreeItem command can be used.

General Methods

Type ( enum key )

Selects the type of Plot.

key can have one of  the following values:

"arrows"

The field vectors will be plotted as arrows.

"cones"

The field vectors will be plotted as cones.

"thinarrows"

The field vectors will be plotted as thin arrows.

"lightarrows"

The field vectors will be plotted as light arrows (cone head and line tail).

"filledlinearrows"

The field vectors will be plotted as line arrows with filled head.

"linearrows"

The field vectors will be plotted as line arrows.

"bubble"

The field vectors will be plotted as bubble.

"hedgehog”

The field vectors will be plotted as ”lines”.

"streamline”

Stream lines (cylindrical shapes) will be used to represent the fields. (Interesting mainly for the representation of pointing vectors)

"thinstreamline”

Stream lines (line shapes) will be used to represent the fields. (Interesting mainly for the representation of pointing vectors)

 

 

SetDistributionType ( enum scale )

GetDistributionType enum

Defines / returns the distribution type used for arrow/bubble plotting. The following types are supported for 3D vector plots: "grid", "element" and "surface".  The default for 3D vector plots is "grid".

"grid"

Rectangular distribution on a cartesian grid in 3D.

"element"

Arrows/bubbles are distributed on mesh elements (cells/tetrahedrons).

"surface"

Arrows/bubbles are distributed on the surface of the 3D object.

 

SetDensity( single percentage )

GetDensity single

Specifies / returns the relative  number of plotted objects. The valid value range is between 0.0 (few arrows) and 100.0 (all arrows). Default density is set to 50.

 

SetSurfaceOffset ( single surfaceOffset )

GetSurfaceOffset single

Sets / returns the distance of arrows/bubbles to distance relative to the maximum arrow/bubble size for "surface" distribution. The valid value range is between -1.0 and 1.0. The default is 0.0.

 

SetZoomAdaptiveDistribution( bool adaptive )

GetZoomAdaptiveDistribution bool

Specifies / returns if view and zoom dependent distribution of arrows/bubbles is en-/disabled. If enabled arrows/bubbles will be redistributed if view/zoom changed. If disabled arrows/bubbles distribution is static.

 

SetMaxSize ( single scale )

GetMaxSize single

Defines / returns a scale factor which is applied on the arrows/bubbles. Final arrow size depends on many other factors. E.g. default, arrows will also be scaled by field value ("Length by value" / "Width by value") and zoom / view.  A value between 0.0 and 100.0 may be given. The default setting is 50.

 

SetMinSize ( single minSize )

GetMinSize single

Sets / returns the minimum size of arrows relative to maximum size. This is only effective when "Length by value" or "Width by value" is active. By default arrows/bubbles will be scaled down to zero size if the field value is zero. A value between 0.0 and 1.0 may be given. The default min size is 0.0. A value of 0.3 means that the smallest arrow is at least 30% of the maximum possible arrow size.

 

SetAspectRatio( single aspectRatio )

GetAspectRatio single

Sets / returns a factor applied on the ratio of arrow length and arrow width. Values smaller than 1.0 will make the arrows wider, while values greater than 1.0 will make the arrows longer. Valid value range is between 0.5 and 2.0. Default is 1.0.

 

SetLengthByValue( bool lengthByValue )

GetLengthByValue bool

Specifies / returns if the arrow length is scaled dependent on the field value. If disabled the arrow will have the length derived from the maximum arrow size. Default is TRUE.

 

SetWidthByValue( bool widthByValue )

GetWidthByValue bool

Specifies / returns if the arrow width is scaled dependent on the field value. If disabled the arrow will have the width derived from the maximum arrow size. Default is TRUE.

 

SetZoomAdaptiveScaling( bool adaptive )

GetZoomAdaptiveScaling bool

Specifies / returns if adaptive scaling is en-/disabled. If enabled arrows/bubbles size is dependent of zoom. This means an arrow will keep the same size in screen space (pixels) independent of zoom level. If "zoom adaptive scaling" is disabled the arrow size will fix in 3D space. Default is TRUE

Streamline Methods

SetSeedingStrategy ( enum strategy )

GetSeedingStrategy enum

The distribution of start points (seed points) for streamlines can be steered in various ways using different seeding strategies. Defines / returns the seeding strategy to be used.

"field"

Field based seeding chooses arbitrary high field values, but not necessarily the maximum field values to seed steamlines.

"model"

The application decides where to seed streamlines according to picked points or faces or model features. The number of seed points can be controled using SetSeedCount. Furthermore, the seed points can be filtered using SetSeedThreshold.

"model_density"

The application decides where to seed streamlines according to picked points or faces or model features. The density of seed point distribution can be controled using SetSeedDensity. Furthermore, the seed points can be filtered using SetSeedThreshold.

"interactive"

The streamlines are seeded on a user defined plane which can be moved interactively.

 

SetSeedCount( int number )

GetSeedCount int

Specifies / returns the number of seed points used to generate streamlines with the Field based and Model based (count) strategy.

 

SetSeedDensity( double number )

GetSeedDensity double

Specifies / returns the density of seed point distribution for Model based (density) strategy given in (length project unit)^-2.

 

SetSeedThreshold( int threshold )

GetSeedThreshold int

If one of the Model based strategies is selected, you can set/get a relative threshold to filter seed points by means of the field value at the seed point's position. All seed points with field values below the given threshold (relative to the maximum seed point field value) are filtered away and do not contribute to the streamline plot. E.g. with 0% no seed points are filtered away, with 100% only the seed point with the maximum field value contributes to the plot.

 

SetStreamLinesWidth( single width )

GetStreamLinesWidth single

Specifies / returns the width of the stream lines. The valid value range is between 0 and 100. Default is 50.

Query Methods

AddListItem ( double x, double  y, double z)

Adds a field value query point to the internal coordinate list. Coordinates are expected in project units.

 

SetPoints( variant xArray, variant yArray, variant zArray )

Sets a list of field value query points to the internal coordinate list. Coordinates are expected in project units.

 

CalculateList

Calculates field values at all query points of the internal coordinate list.

 

GetListLength long

Returns the number of field values calculated by CalculateList.

 

GetListItem( long index, double_ref FieldXRe, double_ref FieldYRe, double_ref FieldZRe, double_ref FieldXIm, double_ref FieldYIm, double_ref FieldZIm ) bool

Returns the field value calculated by CalculateList at position index.

 

GetList( string fieldComponent ) variant

Returns the field values calculated by CalculateList as an array of doubles. Allowed values for fieldComponent are "xre", "yre", "zre", "xim", "yim", "zim".

 

Reset

Clears the internal storage used by the field list evaluation and empties the evaluation point storage.

Default Settings

Type ("arrows")

SetDensity (50)

SetMaxSize (50)

Example

The following script plots the electric field  "e1" (if available) in a linear scale by using  thin arrows with density 30%.

 

' Plot only a wire frame of the structure to be able to look inside

Plot.wireframe (True)

 

' Select the desired monitor in the tree.

SelectTreeItem ("2D/3D Results\E-Field\e1")

 

' Plot the field of the selected monitor

With VectorPlot3D

    .Type ("thinarrows")

    .SetDensity (30)

End With
