LumpedElement Object

Lumped elements can be used to include electronic components such as resistors, inductances, capacitances and diodes into the simulation. Spice or Touchstone files can be also imported (up to 2 pins with the lumped element type and with more than 2 pins with the multipin element type).

  • Methods

  • Defaults

Methods

Reset

Resets all internal values to their default settings.

 

Create

Creates a new lumped element. All necessary settings for this element have to be made previously. For the definition of a multipin lumped element (type multipingroupspice or multipingrouptouchstone) use the CreateMultipin method.

 

CreateMultipin

Creates a new multipin lumped element. All necessary settings for this element have to be made previously. For the definition of elements other than multipin (type multipingroupspice or multipingrouptouchstone) use the Create method.

 

Modify

Modifies an existing element. Only geometrical settings which were made previously are changed.

 

SetType ( enum key )

Sets the type of the element.

enum key

meaning

"rlcserial"

The discrete element is made out of a serial circuit of a resistance, an inductance and a capacitance. (default)

"rlcparallel"

The discrete element is made out of a parallel circuit of a resistance, an inductance and a capacitance.

"short"

The discrete element is made out of a short circuit, this is equivalent to a "rlcserial" element with zero (i.e. missing) resistance, inductance and capacitance.

"open"

The discrete element is made out of an open circuit, this is equivalent to a "rlcparallel" element with zero (i.e. missing) resistance, inductance and capacitance.

"diode"

The discrete element is a diode.

"spicecircuit"

The discrete element is defined by circuit file with Spice format (typical extensions are *.cir, *.net).

"touchstone"

The discrete element is defined by circuit file with Touchstone format (typical extensions are *.ts, *.s*p).

"multipingroupitem"

The discrete element is a multipin sub-element to be connected to a multipin lumped element (with type "multipingroupspice" or multipingrouptouchstone").

"multipingroupspice"

The discrete element is a multipin lumped element defined by circuit file with Spice format (typical extensions are *.cir, *.net).

"multipingrouptouchstone"

The discrete element is a multipin lumped element defined by circuit file with Touchstone format (typical extensions are *.ts, *.s*p).

 

SetName ( name name )

Sets the name of the element.

 

Folder ( name foldername )

Sets the name of the folder for the new element. If the name is empty, then the element does not belong to a folder.

 

SetR / SetL / SetC ( double dValue )

Defines the value of resistance / inductance / capacitance of the element.

 

SetGs / SetI0 / SetT ( double dValue )

Sets the blocking conductivity / reverse current / the temperature in Kelvin for the diode. These methods have only an effect if the .SetType method is set to ”diode”.

 

SetP1 / SetP2 ( bool picked, double x, double y, double z )

Define the starting / end point of the element. If picked is True, two previously picked points will be used for the coordinates of the start / end point. Otherwise the point will be defined by x / y / z.

 

GetCoordinates ( name name, double x0, double y0, double z0, double x1, double y1, double z1  ) bool

Queries the start and end point coordinates of an element specified by name (use colon to separate the folder from the element name).

 

GetProperties ( name name, string  type, double R, double L, double C, double Gs, double I0, double T, double radius  ) bool

Queries the basic values of the element with the given name (use colon to separate the folder from the element name). All other parameters are out-values. Returns false if name do not exists, true otherwise.

 

CircuitFileName ( name filename )

Sets the name of the external Spice or Touchstone file to be imported for the spicecircuit, touchstone, multipingroupspice and multipingrouptouchstone element type.

 

UseRelativePath ( bool flag )

Defines if the CircuitFileName is given with a relative or absolute path. In the former case CircuitFileName must specify a location relative to the current project file.

 

CircuitId ( int Id)

Set a unique id to identify the local copy of the circuit file.

 

UseCopyOnly ( bool flag )

Only the initially copied circuit file in the project folder is used. Any later changes in the external file are ignored.

 

SetInvert ( bool switch )

Set switch to True to reverse the orientation of the element. This might be important, if the lumped element is a diode.

 

SetMonitor ( bool switch )

If switch is True, current and voltage between start and end point of the element will be monitored.

 

SetRadius ( double value )

Specifies a radius for the element.

 

Wire ( string  wirename )

Defines the name of the wire, on which the lumped element is attached to.

 

Position ( string  name )

Defines the end of the wire, on which the lumped element is attached to. Possible values are 'end1' or 'end2'.

 

PortName ( name name )

If a port name is set, the new element will retrieve its geometry definition from an existing discrete port. Geometrical properties defined by commands like SetP1, Wire, Position, SetInvert, ... are ignored. By default, the port is then deleted, so this is used for converting discrete ports into lumped elements.

 

DeletePort ( bool flag )

If a PortName is set, this flag decides, whether the port is deleted after conversion to a lumped element or whether it is kept. The default is "True", so the port will be deleted.

 

 

StartLumpedElementNameIteration ()

Initialize an iterator over all elements and place it before the first element. Returns the current number of elements in the list.

 

ConnectMultipinElementPinToSubElement ( name multipin_name, name circuit_pin_name,  name multipin_sub_element_name  )

Connect a circuit pin of a multipin lumped element (type multipingroupspice or multipingrouptouchstone) to a multipin sub-element (type multipingroupitem).

 

ConnectMultipinElementPinToShort ( name multipin_name, name circuit_pin_name  )

ConnectMultipinElementPinToOpen

Define a circuit pin of a multipin lumped element (type multipingroupspice or multipingrouptouchstone) be connected to the ideal circuit ground (ConnectMultipinElementPinToShort method) or to be kept floating (ConnectMultipinElementPinToOpen method).

 

GetNextLumpedElementName ( string  name ) bool

Returns the name of the next available element. Returns false if there are no other elements, true otherwise. Use GetNextLumpedElementName to initialize the iteration. Example of usage:

 

Dim itemName As String

Dim lType As String  Dim valueR As Double

Dim valueL As Double

Dim valueC As Double

Dim valueGs As Double

Dim valueIO As Double

Dim valueT As Double

Dim valueRadius As Double

 

LumpedElement.StartLumpedElementNameIteration()

Do While LumpedElement.GetNextLumpedElementName(itemName)

If LumpedElement.GetProperties(itemName, lType, valueR, valueL, valueC, valueGs, valueIO, valueT, valueRadius) Then

MsgBox ("Test: " & lType & " " & valueR & " " & valueL & " " & valueC & " " & valueGs & " " & valueIO & " " & valueT & " " & valueRadius )

Else

MsgBox ("Element does not exist")

End If

Loop

 

 

Rename ( name sOldName, name sNewName )

Changes the name of an existing object.

 

Delete ( name name )

Deletes an existing object.

 

NewFolder ( name foldername )

Creates a new folder with the given name.

 

DeleteFolder ( name foldername )

Deletes an existing folder and all the containing elements.

 

RenameFolder ( name oldFoldername, name newFoldername )

Changes the name of an existing folder.
