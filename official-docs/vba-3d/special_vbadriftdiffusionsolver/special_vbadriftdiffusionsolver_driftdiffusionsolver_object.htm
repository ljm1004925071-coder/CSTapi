Drift-Diffusion Solver Object

The Object is used to define the drift-diffusion solver parameters.

  • Methods

  • Default Settings

  • Example

Methods

Reset

Resets all internal settings to their default values.

 

SetSolverMode  (string  solvermode)

Adds a static field of type fieldtype, scaled by factor that will influence the particles' trajectories. If recalculate is set to true, the fields will be recalculated before the tracking solver starts.

Note that a factor of "0" leaves the field unconsidered for the tracking process.

solvermode(string )

meaning

"Poisson"

Solve for the non-linear potential.

"Continuity"

Solve for the non-linear potential and carrier densities. The problem is decoupled.

"Gummel-Map"

Solve the coupled problem of potential and carrier densities iteratively.

"Mesh"

Generates the mesh and stops. No solution is computed.

 

AmbientTemperature   (double  temperature)

Sets ambient temperature in Kelvin.

 

NewtonAccuracy (double  accuracy)

Sets the accuracy of  the Newton method used within the Poisson solver.

 

Accuracy  (double  accuracy)

Defines the accuracy of the coupled iteration to solve the potential and carrier densities.

 

MaxIteration  (int  iteration)

Defines the maximum number of iterations of the coupled computation.

 

EnableNormalization  (bool  normalize)

Activates the normalization of the equation system. If disabled, the solver computes the problem in SI units.

 

TetMeshAdaption  (bool  meshadaption)

Activates the mesh refinement based on the error-estimate of the solution quantities. This option is only available for these two solver modes: "Gummel-Map" and "Mesh".

 

TetCellSizeToDebyeLengthRatio  (double cellsizeratio)

If the adaptive mesh refinement is enabled, this setting allows refining a coarse mesh to this ratio prior to simulation start. A cell is refined until it's characteristic length to Debye length ratio is below this threshold. The Debye length is computed using the local doping density. Please note, that this setting might lead to a fast increase in the number of mesh cells. Please start from a rather well resolved mesh.

 

MonitorJunctionCapacitance  (bool junction_capacitance)

Enables the computation of the junction capacitance using a finite-different approximation.

 

MonitorPlasmaFrequency  (bool plasma_frequency)

Legacy command. Please use monitor definition. Computes the plasma frequency as a result after the solver run. The plasma frequency is depending on the carrier densities.

 

UseMaxNumberOfThreads  (bool parallelization)

Uses all allowed threads for a parallel computation.

 

MaxNumberOfThreads  (int numberofthreads)

Uses up to the defined number of threads.

 

MaximumNumberOfCPUDevices  (int numberofcpus)

Uses up to the defined number of CPUs.

 

UseDistributedComputing  (bool dcomputing)

Enables a DC computation.

 

 

Default Settings

SetSolverMode "Gummel-Map"

AmbientTemperature "300"

NewtonAccuracy "1e-9"

Accuracy "1e-5"

MaxIteration "30"

EnableNormalization "True"

TetMeshAdaption "False"

TetCellSizeToDebyeLengthRatio "0"

MonitorJunctionCapacitance "False"

MonitorPlasmaFrequency "False"

UseMaxNumberOfThreads "True"

MaxNumberOfThreads "1024"

MaximumNumberOfCPUDevices "2"

UseDistributedComputing "False"

 

 

Example

With DriftDiffusionSolver

     .Reset

     .SetSolverMode "Gummel-Map"

     .AmbientTemperature "300"

     .NewtonAccuracy "1e-9"

     .Accuracy "1e-5"

     .MaxIteration "30"

     .EnableNormalization "True"

     .TetMeshAdaption "False"

     .TetCellSizeToDebyeLengthRatio "0"

     .MonitorJunctionCapacitance "True"

     .MonitorPlasmaFrequency "False"

     .UseMaxNumberOfThreads "True"

     .MaxNumberOfThreads "1024"

     .MaximumNumberOfCPUDevices "2"

     .UseDistributedComputing "False"

End With
