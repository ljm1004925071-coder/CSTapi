FarfieldCalculator Object

Overview

The FarfieldCalculator object provides access to farfield results without any need to reconfigure the current plot or without the necessity to trigger frontend plot operations. It is therefore well suited for the use in highly automated workflows like parameter sweeps or optimizer runs.

The configuration of the FarfieldCalculator objects is separate from the current plot settings (i.e. the current settings of FarfieldPlot and FarfieldArray). However, the convenience function ResetAsCurrentPlot provides a quick means to set the configuration flags of the FarfieldCalculator according to the current plot.

In general, the methods of FarfieldCalculator can be classified into several groups: configuration methods, methods that trigger farfield calculations, and methods that read out the results. In the following overview, the configuration methods are subdivided in groups according to the settings that they modify. Some of the settings are important for the actual farfield calculation and others only affect the scaling and are therefore only needed during the readout of farfield results.

Configuration Methods (General)

Reset

Resets the FarfieldCalculator object to its default settings and deletes all results which are currently contained in the object.

 

ResetAsCurrentPlot

Deletes all results currently stored in the FarfieldCalculator object and sets the internal settings of the object according to the configuration of the current farfield plot. It also sets the array settings of the FarfieldCalculator according to the current settings of the FarfieldArray Object.

Configuration Methods (Calculation Settings)

The following configuration methods affect the calculated farfield results.

 

SetAxesType ( enum axesType )

Specifies the alignment of the farfield coordinate system. The following types are available:

 

"xyz"

Alignment with the global coordinate system.

"user"

User defined alignment using the orientation specified by Thetastart and Phistart.

"mainlobe"

Aligns the coordinate system with the main lobe direction (z') and the PolarizationVector (y').

"currentwcs"

The coordinate system is set to the current working coordinate system.

 

Phistart ( double x, double y, double z )

Thetastart ( double x, double y, double z )

Set new origin vectors in global cartesian coordinates for the x'-axis and z'axis. The x'-axis defines the new start of the angle phi, the z'-axis defines the new start of the angle theta. Therefore Phistart is used to set the new x'-axis, Thetastart to set the new z'-axis. The resulting vectors will be normalized to 1. The x'-axis will be orthogonalized to the z'-axis. Please make sure that the axes are not parallel and that the vectors do not have zero length. The y'-axis will be determined automatically.

 

PolarizationVector ( double x, double y, double z )

Set the polarization vector or y'-axis in global cartesian coordinates. You may provide a non-normalized vector, it will be automatically normalized to 1.

 

Origin ( enum originType )

The origin type of the farfield calculation. originType can have one of the following values:

 

"bbox"

The center of the bounding box of the structure

"zero"

Origin of coordinate system

"free"

Any desired point defined by Userorigin

 

Userorigin ( double x, double y, double z )

Sets origin of the farfield calculation if the origin type is set to free.

 

IncludeUnitCellSidewalls ( bool flag )

This method affects the farfield calculation for periodic and unit cell boundaries. If flag is true, the farfield integration considers periodic and unit cell boundaries like open boundaries. Otherwise, these side wall contributions are ignored, and the aperture field comprises the open and Floquet port boundaries only.

 

UseArray ( bool flag )

Activates the array pattern for farfield calculation. Note that the array settings of FarfieldCalculator are disjoint from the settings of FarfieldArray. Use ResetAsCurrentPlot to set the settings in line with the current plot settings.

 

Arraytype ( enum arrayType )

Sets the type of the array pattern. arrayType can have one of the following values:

 

"rectangular"

A rectangular array pattern (linear: 1D, planar: 2D, cubic: 3D) can be defined by setting the corresponding antenna numbers together with the space and phaseshift information using the methods ArrayXSet, ArrayYSet, and ArrayZSet. If unit cell or periodic boundaries are active, the array setup sets the unit cell geometry and the phase differences automatically.

"edit"

This mode offers the possibility to edit the current antenna list by adding single antenna elements to the list using the method ArrayAddToList or clearing the list by ArrayClearList.

 

ArrayXSet ( long number, double spaceshift, double phaseshift )

ArrayYSet ( long number, double spaceshift, double phaseshift )

ArrayZSet ( long number, double spaceshift, double phaseshift )

Defines a linear array pattern in the x-,y- or z-direction respectively. Thus a linear, planar or cubic antenna array structure may be created.

 

number

The number of single antenna elements in x-,y- or z-direction

spaceshift

The constant spaceshift between two single antenna elements in x-,y- or z-direction

phaseshift

The constant phaseshift between two single antenna elements in x-,y- or z-direction

 

ArrayAddToList ( double x, double y, double z, double amplitude, double phase )

Defines the properties of a single antenna element. This method is only applicable if the mode set by Arraytype is set to “edit“.

 

x

The x-position of the single antenna element in space

y

The y-position of the single antenna element in space

z

The z-position of the single antenna element in space

amplitude

The amplitude value of the single antenna element

phase

The phase value of the single antenna element

 

ArrayAutoPhaseFromBoundary ( bool flag )

If flag is true, this activates the automatic configuration of the phase shift from the boundary condition setup. This only works for projects with periodic or unit cell boundary conditions and is ignored in other cases.

 

ArrayClearList

Clears the currently set list of array elements.

Configuration Methods (Result Scaling)

The following methods affect the scaling of results retrieved by GetList/GetListItem methods.

 

SetScaleLinear ( bool flag )

If activated the retrieved farfield results are returned in linear scale. Otherwise the scaling is logarithmic.

 

DBUnit ( enum unitCode )

Sets the unit for the retrieved results. This setting affects the results in linear as well as in logarithmic mode. unitCode can have one of the following values:

 

"0"

V/m, A/m, W/m2

dBV/m, dBA/m, dBW/m2

"60"

mV/m, mA/m, uW/m2

dBmV/m, dBmA/m, dBpW/m2

"120"

uV/m, uA/m, pW/m2

dBuV/m, dBuA/m, dBpW/m2

"-60"

kV/m, kA/m, MW/m2

dBkV/m, dBkA/m, dBMW/m2

 

SetLogNorm ( double norm )

Sets the logarithmic normalization, i.e., the reference value for quantities returned in dB. Set to zero (the default value) to turn off manual logarithmic normalization.

 

SetInverseAxialRatio ( bool flag )

If activated the inverse IEEE axial ratio is returned.

 

SlantAngle ( double angle )

Sets the slant angle to angle. This settings is only required for the polarization type "slant".

Calculation Methods

AddListEvaluationPoint ( double polarAngleInDegree, double lateralAngleInDegree, double radiusInMeters,  enum coordinateSystem, enum tf_type, double freq_time )

Defines an evaluation point for CalculateList. The evaluation point is specified by a polar angle and a lateral angle referring to a coordinate system coordinateSystem, which may be one of the following:

"spherical"

Spherical coordinates

"ludwig2ae"

Ludwig2 coordinates - azimuth over elevation

"ludwig2ea"

Ludwig2 coordinates - elevation over azimuth

"ludwig3"

Ludwig3 coordinates

The radius specifies the distance at which the farfield or nearfield should be evaluated. It is expected in meters. Use tf_type = "time" to provide an evaluation time or tf_type = "frequency" to pass an evaluation frequency via freq_time. The latter two settings are only necessary for broadband farfield monitor results and should be left empty for other farfields.

 

AddListEvaluationPoints ( array(string) polarAngleInDegreeArray, array(string) lateralAngleInDegreeArray, array(string) radiusInMetersArray, array(string) coordinateSystemArray, array(string) tf_typeArray, array(string) freq_timeArray )

Similar to AddListEvaluationPoint, this method defines evaluation points for the list calculation. Using this method, many points are added to the list at once. To add N points at a time, all the arrays that are passed in the parameters need to be either of length N or of length 1. If the respective parameter array is of length 1, then this parameter value is assumed to be the same for all the N points. The arguments are expected to be string arrays that are convertible to the respective data types documented for AddListEvaluationPoint. Note that the use of this method is often much faster than the repeated call of AddListEvaluationPoint since it avoids the overhead that comes with a large number of function calls (especially if the template/macro is run from another process, e.g., from DesignStudio or from Python).

 

CalculateList ( string treepath, enum calculationParameters )

Calculates farfield values at the evaluation points defined by the AddListEvaluationPoint command. treepath should be a string that contains the path in the result tree of the farfield result for which the list should be evaluated. calculationParameters is a string that may contain the following tokens separated by space: “farfield”, “nearfield”, “Eonly”, “Honly”.

The tokens “farfield” or “nearfield” specify whether the farfield calculation should be done with or without the farfield approximation. Exactly one of these two tokens must always be present.

By default, the fields in the list are evaluated in such a way that one may freely choose any quantity of interest, e.g. “directivity” or “efield”, at the result retrieval stage later on (see “resulttype” in the documentation of GetList/GetListItem below). However, for nearfield computations this comes with a performance penalty. Therefore, one may explicitly opt out of this default mode by specifying the token “Eonly” or “Honly”. With “Honly”, one may later on exclusively access the “hfield” result type, while “Eonly” provides exclusive access to all other result types.

Note that if the axes setting configured by SetAxesType is set to “mainlobe” each point in the list is separately aligned to its own mainlobe. If the list contains a mixture of nearfield samples at different distances or points at different frequency, this may imply that a repeated mainlobe recomputation will be performed which may be computationally demanding. If instead an alignment to a fixed mainlobe direction is required, consider calculating the mainlobe for the required scenario beforehand and setting it as a fixed coordinate system using Phistart and Thetastart before calling CalculateList.

 

ClearList

ClearList empties the current evaluation list including associated results.

 

GetList ( enum resultType, enum fieldComponent ) variant

GetListItem ( long index, enum resultType, enum fieldComponent ) double

These two commands allow to retrieve the results from a previous CalculateList command. The overall workflow for using these commands is the following:

First, all points have to be added using AddListEvaluationPoint. After a subsequent call to CalculateList, the results can be inquired using the GetListItem command, providing the 0-based index of the points in the same order as they were added. With GetList the whole list of results can be retrieved at once, which is faster than GetListItem in most cases. It will return an Array of double values. Depending on the fieldComponent identifier, the point coordinates or the farfield/nearfield results can be returned.

Use ClearList or Reset to delete the point list.

The actual data to be returned may be chosen by the two identifiers resultType and fieldComponent.

resultType may be one of the following choices:

"directivity"

Returns the directivity

"gain"

Returns the gain

"realized gain"

Returns the realized gain

"efield"

Returns the electric field

"epattern"

Returns the electric field pattern

"hfield"

Returns the magnetic field

"pfield"

Returns the power pattern

"rcs"

Returns the radar cross section (square meters)

"rcsunits"

Returns the radar cross section (project length units squared)

"rcssw"

Returns the radar cross section (square wavelength)

fieldComponent is a concatenation of four component specifiers separated by space:

fieldComponent = <Coord.System> +" " + <Polarization> +" " + <Component> +" " + <ComplexComp.>

Allowed values are:

<Coord.System>

"spherical", "ludwig2ae", "ludwig2ea", "ludwig3"

<Polarization>

"linear", "circular", "slant", "abs"

<Component>

"radial",

"comp1", "theta", "azimuth", "left", "alpha", "horizontal", "crosspolar",

"comp2", "phi", "elevation", "right", "epsilon", "vertical", "copolar"

<ComplexComp.>

"abs", "phase", "re", "im"

The absolute value of the field vector is accessible via:

fieldComponent = <Coord.System> +" " + "abs"

The axial ratio and the component ratios are accessible through

fieldComponent = <Coord.System> +" " + <Polarization> +" " + "axialratio"

fieldComponent = <Coord.System> +" " + <Polarization> +" " + "compratio12"

fieldComponent = <Coord.System> +" " + <Polarization> +" " + "compratio21"

where "compratio12" refers to comp1/comp2.

In addition, the identifier fieldComponent may have one of the following values:

"Point_T"

The spatial theta value of the defined point

"Point_P"

The spatial phi value of the defined point

"Point_R"

The spatial radius value of the defined point.

 

CalculateMainlobe ( string treepath, double radiusInMeters,  enum tf_type, double freq_time )

Calculates the mainlobe result for a farfield specified by its treepath. If the mainlobe of a nearfield is desired, the radius (expected in meters) specifies the distance at which the mainlobe should be calculated. If this radius is set to 0, the mainlobe of the farfield is computed. Use tf_type = "time" to provide an evaluation time or tf_type = "frequency" to pass an evaluation frequency via freq_time. The latter two settings are only necessary for broadband farfield monitor results and should be left empty for other farfields. The result of this calculation can be retrieved by a call to GetMainLobeVector.

 

GetMainLobeVector ( double_ref x, double_ref y, double_ref z )

Returns the direction of the 3D farfield main lobe that was calculated in a preceding CalculateMainlobe call

 

CalculatePhaseCenter ( string treepath, enum{"theta", "phi", "boresight"} component, enum{"both", "e-plane", "h-plane"} plane, double angleInDegrees,  double radiusInMeters,   double userFreq )

Calculates the phase center result for a farfield specified by its treepath. component selects the desired farfield component which is used for the phase center calculation. "theta" and "phi" use the phase of the corresponding spherical components. "boresight" evaluates the farfield in +z' direction, extracts the polarization vector from this field and uses this reference to calculate the phase.

The phase center is calculated in either the E-plane (x' = 0), H-plane (y' = 0) or in both planes. The desired planes may be set by the plane parameter.

The phase center is calculated using the phase values around the z'axis in the x' = 0 (E-Plane) and y' = 0 (H-Plane) planes with a constant distance from the origin. To limit the area around the z'-axis taken into account for the calculation in those planes you may specify an angle using the angleInDegrees parameter.

If the phase center of a nearfield is desired, the radius (expected in meters) specifies the distance at which the nearfield should be calculated. If this radius is set to 0, the phase center of the farfield is computed. Use userFreq to provide an evaluation frequency, which is only necessary for broadband farfield monitor results and should be left empty for other farfields.

The result of this calculation can be retrieved by a call to GetPhaseCenterResult.

 

GetPhaseCenterResult ( enum{"x", "y", "z"} component, enum{"avg", "e-plane", "h-plane"} mode ) double

Returns the phase center location, which was previously calculated by calling CalculatePhaseCenter, in global coordinates for a given component and mode. The component may be either "x", "y" or "z". The mode may be either "avg" (for averaged phase center location), "e-plane" (for the phase center location calculated in the E-plane) or "h-plane" (for the phase center location calculated in the H-plane).

 

GetPhaseCenterSigma ( enum{"x", "y", "z"} component, enum{"avg", "e-plane", "h-plane"} mode ) double

Returns the standard deviation of the Phase Center, which was previously calculated by calling CalculatePhaseCenter, in global coordinates for a given component and mode. The component may be either "x", "y" or "z". The mode may be either "avg" (for averaged phase center location), "e-plane" (for the phase center location calculated in the E-plane) or "h-plane" (for the phase center location calculated in the H-plane).

 

GetTRP ( string treepath,  enum tf_type, double freq_time ) double

Returns the radiated power of a farfield specified by its treepath. Use tf_type = "time" to provide an evaluation time or tf_type = "frequency" to pass an evaluation frequency via freq_time. The latter two settings are only necessary for broadband farfield monitor results and should be left empty for other farfields.

 

GetTotalRCS ( string treepath,  double frequency,  enum{"m2", "units", "sw"} csunit  ) double

GetTotalRCS ( string treepath,  double frequency,  enum{"m2", "units", "sw"} csunit  ) double

Returns the Total RCS/ACS of a farfield specified by its treepath. Use frequency to pass an evaluation frequency. This setting is only necessary for broadband farfield monitor results and should be left empty for other farfields. csunit specifies the unit of the expected result with

"m2"

The RCS/ACS unit is m^2

"units"

The RCS/ACS unit is (project length unit)^2

"sw"

The RCS/ACS unit is (wavelength)^2

 

GetRadiationEfficiency ( string treepath,  double frequency ) double

GetTotalEfficiency ( string treepath,  double frequency ) double

GetSystemRadiationEfficiency ( string treepath,  double frequency ) double

GetSystemTotalEfficiency ( string treepath,  double frequency ) double

Returns the Total RCS/ACS of a farfield specified by its treepath. Use frequency to pass an evaluation frequency. This setting is only necessary for broadband farfield monitor results and should be left empty for other farfields..

 

GetFrequency ( string treepath ) double

Returns the frequency in project units associated with the farfield result specified by its treepath. It returns NAN for broadband farfield monitor results.

 

IsBroadbandMonitor ( string treepath ) bool

Returns True if the farfield result specified by its treepath is a broadband farfield monitor result or False otherwise.

Coordinate Transformation Methods

CartesianToSpherical ( enum coordinateSystem, double x, double y, double z ) variant (array containing radius, polarInDegrees, lateralInDegrees)

SphericalToCartesian ( enum coordinateSystem, double radius, double polarInDegrees, double lateralInDegrees ) variant (array containing x, y, z)

These functions convert from cartesian coordinates to spherical coordinates (CartesianToSpherical) or vice versa (SphericalToCartesian). The cartesian direction vector is converted to one of the following spherical coordinate systems, which can be set by the coordinateSystem string with the following allowed values:

"spherical"

Spherical coordinates

"ludwig2ae"

Ludwig2 coordinates - azimuth over elevation

"ludwig2ea"

Ludwig2 coordinates - elevation over azimuth

"ludwig3"

Ludwig3 coordinates

In the function signature, radius denotes the radial distance, polarInDegrees denotes the polar angle of the coordinate system (e.g. Theta, Elevation, or Alpha) in degrees and lateralInDegrees denotes the lateral angle of the coordinate system (e.g. Phi, Azimuth, or Epsilon) in degrees.

 

Note that these functions do pure spherical to cartesian conversions and do not take into account any coordinate system settings of FarfieldCalculator. If in addition a conversion between global and local coordinates is required, consider combining this function with the CartesianGlobalToLocal/CartesianLocalToGlobal functions.

In VBA, the functions return a 3-element Variant array that contains the double values. In Python, they are returned as a tuple.

 

CartesianGlobalToLocal ( double x_global, double y_global, double z_global ) variant (array containing x_local, y_local, z_local)

CartesianLocalToGlobal ( double x_local, double y_local, double z_local ) variant (array containing x_global, y_global, z_global)

These functions convert cartesian direction vectors from global to local coordinates (CartesianGlobalToLocal) or vice versa (CartesianLocalToGlobal). The local coordinate system that is used for the transformation is configured by the SetAxesType and the Phistart/Thetastart methods. Note that the axes type "mainlobe" is not supported for coordinate transformations.

In VBA, the functions return a 3-element Variant array that contains the double values. In Python, they are returned as a tuple.

Arraytype (“rectangular”)
