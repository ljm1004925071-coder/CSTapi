Fan Object

This object is used to create new fan object for the Conjugate Heat Transfer Solver.

  • Methods

  • Default Settings

  • Example

Methods

Reset

Resets all internal settings of the fan source to their default values.

 

Name ( string name )

Specifies the name of the fan.

 

Folder ( name foldername )

Sets the name of the folder for the new fan. If the name is empty, then the fan does not belong to a folder.

 

Enable( bool flag )

Enables or disables the fan being created. If flag is false, the fan won't be considered for simulation and the underlying geometry along with the material will be considered instead.

default: value = True

 

EnableItem( name  name,  bool flag )

Enables or disables the (existing) fan with the given name depending on flag. If flag is false, the fan will not be considered for simulation.

 

FlowType ( enum type )

Selects the type of the flow description for the fan. Currently only the type "Normal" is supported.

 

ConstructionType ( enum type )

Specifies the fan construction type. Currently two types are available:

"axial"

One entry and one exit face. They must belong to the same shape and they must be parallel. Only one entry, or only one exit face might be defined for thin fans.

"centrifugal"

Multiple entry or exit faces. They must belong to the same shape

 

default: type = "axial"

 

InvertFlowDirection( bool flag )

Enables or disables the flow direction being inverted. If flag is True, the flow direction will be inverted with respect to its original definition.

default: value = False

 

DeratingFactor ( double expression value )

Sets the derating factor for the fan.

default: value = 1.0

 

FlowSpecType ( enum type )

Specifies whether the temperature or dissipated heat will be prescribed for the fan. Available types are "Dissipated heat" and "Temperature".

default: type = "Dissipated heat"

 

DissipatedHeat ( double expression value )

Sets the dissipated heat value for the fan in Watt. This setting will be considered by the solver only, if the flow specification type is "Dissipated heat".

default: value = 0

 

Temperature ( double expression value, enum unit  )

Sets the temperature value for the fan in the given unit.  Available units are "Celsius", "Fahrenheit" and "Kelvin". This setting will be considered by the solver only, if the flow specification type is "Temperature".

default: value = 0, unit = <project temperature unit>

 

FanCharacteristics ( enum type )

Specifies which parameters will be provided to define the fan characteristics. Available fan characteristic types are:

"FixedVolume"

The volume flow rate and the corresponding unit are taken into account.

"Linear"

The volume flow rate and the stagnation pressure with respective units are taken into account.

"Nonlinear"

The fan characteristics are described by a user-defined Flow rate / Pressure curve.

 

default: type = "FixedVolume"

 

VolumeFlowRate ( double expression value )

Specifies the volume flow rate. This setting is ignored for the "nonlinear" fan characteristics type.

default: value = 0

 

VolumeFlowUnit ( enum unit )

Specifies the unit in which the volume flow rate or the fan curve abscissa is given. Available units are "m3/h" and "CFM".

default: unit = "m3/h"

 

StagnationPressure ( double expression value )

Specifies the stagnation pressure for the fan. This setting is considered only for the "linear" fan characteristics type.

default: value = 0

 

PressureUnit ( enum unit )

Specifies the unit in which the stagnation pressure or fan curve pressure is prescribed. Available units are "atm", "bar", "Pa" and "psi". This setting is ignored for the "FixedVolume"" fan characteristics type.

default: unit = "Pascal"

 

AddCurveValue ( double expression flow_rate, double expression pressure )

Adds a (flow_rate, pressure) pair to the user-defined nonlinear fan curve definition. At least two pair have to be defined to create a curve. This setting is considered only for the "nonlinear" fan characteristics type.

 

AddEntryFace ( solidname solidname, int faceid  )

Defines the face on which the flow enters the fan. This will be the face belonging to the specified faceid of the solid with name solidname.

 

AddExitFace ( solidname solidname, int faceid  )

Defines the face on which the flow leaves the fan. This will be the face belonging to the specified faceid of the solid with name solidname.

 

Create

Creates the fan with its previously made settings.

 

Delete ( string name )

Deletes the fan with the given name.

 

Rename  ( string oldname,  string newname  )

Renames the fan with the given oldname to the defined newname.

 

NewFolder ( string foldername )

Creates a new folder with the given name.

 

RenameFolder ( string oldFoldername, string newFoldername)

Changes the name of an existing folder.

 

DeleteFolder ( string foldername )

Deletes an existing folder and all the contained elements.

 

UseMLFanLibrary( bool flag )

Enables or disables the creation of synthetic non-linear fan curve by using a trained Machine Learning (ML) model. It has effect only if FanCharacteristics is "Nonlinear".

default: value = False

 

MLFanFrameSize( double expression value, enum unit  )

Sets the fan frame size for the ML curve generation in the given unit.  Available units are "mm", "cm" and "in". This setting will be considered by the solver only, if UseMLFanLibrary is True.

default: value = 30, unit = "mm"

 

MLFanThickness( double expression value, enum unit  )

Sets the fan thickness for the ML curve generation in the given unit.  Available units are "mm", "cm" and "in". This setting will be considered by the solver only, if UseMLFanLibrary is True.

default: value = 30, unit = "mm"

 

MLFanRotSpeed( double expression value, enum unit  )

Sets the fan rotational speed for the ML curve generation in the given unit.  Available units are only  "rpm". This setting will be considered by the solver only, if UseMLFanLibrary is True.

default: value = 3000, unit = "rpm"

 

MLFanMaxFlowRate( double expression value, enum unit  )

Sets the fan maximum flow rate for the ML curve generation in the given unit.  Available units are "CFM", "m^3/h " and "l/s ". This setting will be considered by the solver only, if UseMLFanLibrary is True.

default: value = 2, unit = "CFM"

 

MLFanMaxPressure( double expression value, enum unit  )

Sets the fan maximum pressure for the ML curve generation in the given unit.  Available units are "mmH2O ", "Pa ", "Bar", "psi", "Torr", "mmHg",  and "inH2O ". This setting will be considered by the solver only, if UseMLFanLibrary is True.

default: value = 65.5, unit = "mmH2O "

 

ActivateSwirlingExitFlow( bool flag )

Enables or disables the swirling exit flow. It only has an effect if the fan's flow exit direction points towards the domain. This is determined by  the fan's exit face and the invert flow direction flag.

default: value = False

 

FanSpeed( double expression value )

Specifies the speed of the fan rotation in RPM. This setting is considered only if ActivateSwirlingExitFlow is True.

default: value = 5000

 

FanSwirlMagnitude( double expression value )

Specifies the ratio of the swirl velocity in the direction of revolution to the axial velocity . This setting is considered only if ActivateSwirlingExitFlow is True.

default: value = 0.05

 

FanRotationDirection ( enum direction)

Specifies the direction of the fan. Available options are "Counterclockwise" and "Clockwise". This setting is considered only if ActivateSwirlingExitFlow is True.

default: direction= "Counterclockwise"

 

SpeedType( enum type)

Specifies how the fan speed is prescribed. It can have two values: "Fixed Speed", for a constant derating factor, or "Dynamic Speed", where the derating factor depends on the temperature at a monitor, specified by TemperatureReferenceName. This setting is considered only if the CHT solver is set to transient mode.

default: direction= "Fixed Speed"

 

AddTrCurveValue( double expression temperature, double expression derating_factor)

Adds a (temperature, derating_factor) pair to the user-defined dynamic speed curve definition. At least two pair have to be defined to create a curve. This setting is considered only for the "Dynamic Speed" SpeedType type.

 

TrCurveInterpolation( bool flag )

Enables or disables curve interpolation for the fan dynamic speed curve. If False, a step-wise profile is used, if True, linear interpolation is used instead. This setting is considered only for the "Dynamic Speed" SpeedType type.

default: value = False

 

TrCurveTemperatureUnit( enum unit )

Specifies the unit in which the fan dynamic speed curve is prescribed  Available units are "Celsius", "Fahrenheit" and "Kelvin". This setting is considered only for the "Dynamic Speed" SpeedType type.

default: unit = <project temperature unit>

 

TemperatureReferenceName( string monitor_name)

Sets the name of the monitor which will be used as a reference for the computation of the dynamic derating factor. This setting is considered only for the "Dynamic Speed" SpeedType type.

Example

With Fan

     .Reset

     .Name "myfan"

     .FlowType "Normal"

     .DeratingFactor "1"

     .FlowSpecType "Temperature"

     .Temperature "20", "Celsius"

     .FanCharacteristics "Linear"

     .VolumeFlowUnit "m3/h"

     .PressureUnit "Pa"

     .VolumeFlowRate "0"

     .StagnationPressure "0.2"

     .AddEntryFace "component1:solid5", "4"

     .AddExitFace "component1:solid5", "6"

     .Create

End With
