GeneralCompactThermalModel Object

This object is used to create new  general compact thermal model sources for the Conjugate Heat Transfer Solver.

The general compact thermal model can be used to approximate the thermal behavior of single-die packages that can be effectively represented by a thermal resistance network. It is a generalization of the Delphi compact thermal model to arbitrary geometry and thermal network configuration.

Methods

Reset

Resets all internal settings to their default values.

 

Name ( name  name )

Sets the name of the general compact thermal model source. Each source must have a unique name.

 

Rename ( name  oldname,  name newname )

Renames the general compact thermal model with the given oldname to the defined newname. A model with the new name must not already exist.

 

Folder ( name foldername )

Sets the name of the folder for the general compact thermal model under creation. The folder will be created if it does not exist.  If the name is empty, then the general compact thermal model does not belong to a folder.

 

NewFolder ( name foldername )

Creates a new folder for general compact thermal models with the given name. The name of the new folder (foldername) must be unique. Otherwise no new folder will be created.

 

RenameFolder ( name  oldname,  name newname )

Renames an existing general compact thermal model folder from oldname to newname, if a folder with newname does not already exist. All items below the renamed folder will remain below the folder with the new name.

 

DeleteFolder ( string foldername )

Deletes an existing folder and all the contained elements.

 

Enable( bool flag )

Enables or disables the compact thermal model under creation depending on flag. If flag is false, the compact thermal model won't be considered for simulation.

 

EnableItem( name  name,  bool flag )

Enables or disables the (existing) compact thermal model with the given name depending on flag. If flag is false, the compact thermal model won't be considered for simulation.

 

Delete ( name  name )

Deletes the compact thermal model with the given name.

 

Set ( string parameterName, variant parameterValues, ... )

To define special parameters for a compact thermal model, the command Set can be used. Available parameterNames are listed below. All settings must be made before the Create command is called.

parameterName

parameterValues

Default

Meaning

"Name"

name itemName

<empty / invalid>

Specifies the name of the general compact thermal model under creation (same as ".Name" command)

"Face"

solidname solidName, int faceId, int nodeIndex

<empty / invalid>

Assigns a face of a given solid  to the node with the given nodeIndex.  The solid is specified by solidName and the face is indicated by faceId which is the id of the face index in the list of faces belonging to the specified solid.

"NetworkNodeName"

int nodeIndex, string nodeName

<empty / invalid>

Defines the name of the node with the given nodeIndex.

"NetworkNodeType"

int nodeIndex, enum type

{ "Internal", "External", "Leads" }

<1>, "Internal"

<all other nodes>, "External"

Specifies the type of the node with the given nodeIndex.

If the type Internal is specified, the node is an internal node, which participates in heat transfer only via the thermal resistance network.

If the type External is specified, the is an external node, which participates in heat transfer in the compact thermal model via the thermal resistance network and to the environment.

If the type Leads is specified, the is an external node, which participates in heat transfer in the compact thermal model via the thermal resistance network and to the environment, but does not participate in convective heat transfer.

"NetworkNodeCapacitance"

int nodeIndex, double nodeCapacitance

0

Defines the capacitance of the node with the given nodeIndex.

"NetworkNodePower"

int nodeIndex, double nodePower

0

Defines the power of the node with the given nodeIndex. The nodePower is ignored for non-internal nodes.

"NetworkNodeOverwriteEmissivity"

 

"NetworkNodeEmissivity"

int nodeIndex, bool flag

 

int nodeIndex, double emissivity

<all nodes>, False

 

<all nodes>, 0.0

 When enabled (flag = True), the user-defined emissivity is used to specify the surface emissivity of the node with the given nodeIndex. Otherwise (flag = False), the emissivity of the material assigned to the package (Thermal) is used and the value of "NetworkNodeEmissivity" is ignored. These settings are ignored for internal nodes.

"NetworkNodeOverwriteAbsorptance"

 

"NetworkNodeAbsorptance"

int nodeIndex, bool flag

 

int nodeIndex, double absorptance

<all nodes>, False

 

<all nodes>, 0.0

 When enabled (flag = True), the user-defined absorptance is used to specify the surface solar absorptance of the node with the given nodeIndex. Otherwise (flag = False), the emissivity of the material assigned to the package (Thermal) is used and the value of "NetworkNodeAbsorptance" is ignored. These settings are ignored for internal nodes.

"NetworkResistance"

int nodeIndex1, int nodeIndex2, double resistance

<unset / no connection>

Defines the resistance between the node pair (nodeIindex1, nodeIndex2). The node / resistance table is assumed to be symmetric, so it is not necessary to repeat the same settings for node pair (nodeIindex2, nodeIndex1). If redundant settings are made, the last one wins over the previous ones.

 

 

Create

Creates a general compact thermal model source. All required settings for this object have to be made before the Create command is called and will be checked for correctness and consistency during creation.

 

DeleteItemNode ( name  itemname,  name nodename )

Deletes the node nodename of item itemname and clears all corresponding geometry assignments.

 

RenameItemNode ( name  itemname,  name oldnodename,  name newnodename )

Renames the node oldnodename of item itemname to newnodename.

 

ClearFaceList

Clears the current face list used for node geometry assignment changes.

 

ClearNodeFaces ( name  itemname,  name nodename )

Clears all geometry assignment for the node nodename of item itemname.

 

AddNodeToFace ( name  itemname,  name nodename,  solidname solidName, int faceId )

    Adds an entry to the face list to add the face number faceId from shape solidName to the node nodename of item itemname. Use StoreFaces to apply the changes.

 

StoreFaces

Applies all face assignments in the face list to their respective items and nodes.
