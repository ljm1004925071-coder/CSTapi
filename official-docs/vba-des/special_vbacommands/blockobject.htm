General Methods

Create

Creates a new block. Before a block's creation, its Type has to be specified. Some block types require additional properties to be set before the block creation (e. g. a file name). If block's the name has not been set, a default name for the specified block type will be used.

 

Delete

Deletes the currently selected block.

 

DoesExist bool

Checks if a block with the currently selected name already exists.

 

Reset

Resets all internal settings to their default values.

Identification

Name ( name blockname )

This method is used to select an existing block of your model using its name. Furthermore, it sets the name of a block to blockname if not created yet.

Getter

 GetTypeName string

Returns the "type" name of a block. For example the type name of a resistor would be "resistor".

 

GetTypeShortName string

Returns the short "type" name of a block. For example the short type name of a resistor would be "RES".

 

GetFile string

Returns the name of the file associated with the block. (For blocks without file the string is empty.)

 

GetFilePath( int index ) string

Returns the name of the file associated with the block for the given file index.

 

GetNumberOfFiles int

Returns the number of files used by the block.

 

GetSpiceFilter string

This method can only be applied to SPICE- or SPICE file blocks. It returns the SPICE dialect being applied to the block.  For possible return values see the description of the SetSpiceFilter command.

 

GetSpiceModel string

This method can only be applied to SPICE- or SPICE file blocks. It returns the name of the currently selected Spice model aka toplevel subcircuit of the Spice file that is represented by the block.

 

GetPackage string

 It returns the package name specified in the selected block. For possible return values see the description of the SetPackage command. If no package was specified the function returns an empty string.

 

GetAllPackages variant

This method returns all possible package names specified for the selected block. The query can be used as an input for the SetPackage command.

 

GetPackageLengthMM double

 It returns the package length of the selected block in mm. If no package was specified the function returns zero.

 

GetPackageWidthMM double

 It returns the package width of the selected block in mm. If no package was specified the function returns zero.

 

GetPackageHeightMM double

 It returns the package height of the selected block in mm. If no package was specified the function returns zero.

 

GetReferenceName string

Returns the name of block's reference block (or an empty string if the block has no reference block).

 

GetNextUnusedName ( name basename ) string

searches the model for the named blocks 'base name + index' and returns the block name with this base + first index that is not used.

 

IsCachingFiles bool

Returns true if the block is caching its referenced files, otherwise false. Works only for blocks of type Touchstone file block.

 

GetSimulationTask string

Returns the referenced simulation task for simulation project reference blocks and hybrid solver reference blocks.

Setter

Type ( string type )

Sets the type for a block. This setting must be made before calling Create. The type can not be modified after the creation. The list of block types contains all valid options.

 

SetName ( name blockname )

Modifies the name of an existing block.

 

SetFile ( filename filename )

Sets the file for a block. This is only possible for block types that reference external data (file blocks) and for project blocks. In case of project blocks the specified file is copied into the project directory.

 

AddFile( filename filename )

Appends a file to the list of files used by the block. Works only for blocks of type Touchstone file block/Touchstone block.

 

ChangeFile( int index, filename filename )

Modifies the file entry of the given index in the list of files used by the block.

 

RemoveFile( int index )

Removes the file with the given index from the list of files used by the block. Works only for blocks of type Touchstone file block/Touchstone block.

 

SetGlobalUnitForProperty ( name propertyname )

Sets the global unit for the specified property.

 

SetLocalUnitForProperty ( name propertyname, string unit )

Sets the given unit for the specified property, i.e. the property's value will not refer to the project's global unit any more but to this local unit.

 

SetLibraryModel ( name model )

Sets the library model for a block. This is only possible for block types that support the device model library. The name is the tree path in the device model library which is also displayed in the "Model name" field of the block property page. See also ImportLibraryModel command if you want to import a new model.

 

SetRelativePath ( bool relative )

Specifies a relative file name. This is only possible for some types of blocks.

 

SetSimulationModel ( enum model,  bool othersolverincaseoffailure )

Sets the simulation model for a block.
This is only possible for some types of blocks.

Set othersolverincaseoffailure if you don't want the simulation to stop if the selected solver is not supported for the current parameter set.
(only supported for some microstrip and stripline blocks)

 

model may have the following settings:

"Standard"

Standard model

"Fast2D"

Fast 2D EM based model
(only available for some microstrip and stripline blocks)

"2D"

Enhanced 2D EM based model
(only available for microstrip and stripline blocks)

"3D"

3D calculation performed by CST Microwave Studio
(only available for microstrip, stripline and waveguide blocks)

"Dispersive"

Dispersive model
(only available for CS, PCBS and ideal transmissionline blocks)

"Nonlinear"

Nonlinear model based on state space interpolation
(only available for EMS blocks with M-Statics solver option)

"3D_Combined_Cable"

Enables fully coupled cable - 3D EM field simulation.
(only available for CST Cable Studio blocks)

 

SetSpiceFilter ( string type )

This method can only be applied to SPICE- or SPICE file blocks.

It specifies the SPICE dialect being applied. Currently, the types SPICE3, PSPICE, HSPICE, AUTOMATIC are supported. AUTOMATIC  is a combination of SPICE3 (version 3f4) and a subset of PSPICE, where SPICE3 takes precedence over PSPICE, if conflicts occur.

 

SetSpiceModel ( string model )

This method can only be applied to SPICE- or SPICE file blocks. It specifies the Spice model aka subcircuit of the Spice file that is to be represented by the block. This subcircuit is selected and the schematic is updated according to the possibly changed block pins. "model" is the name of a toplevel subcircuit (i.e., not a local subcircuit) of the Spice netlist. On non-existence, an error is generated.

Note that the available Spice models can be queried via the VBA method GetToplevelSubcircuits.

Note that this method needs to be called after block creation. The method is useful to select individual Spice models of Spice files that contain more than one subcircuit.

 

SetPackage ( string type )

This method specifies the package of a block used during the assembly of complex structures in the "Assembly View". It can only be applied to blocks, which have the possibility to carry package information, for example, circuit elements or Touchstone blocks. Please refer to the command GetAllPackages in order to get an overview of the available packages. In general "type" corresponds to the entries of the "Package" drop-down list within the block parameter list. To unset a package please hand over an empty string or "None".

 

SetReferenceBlock ( name blockname )

Sets the reference block for a block. This is only possible for some types of blocks.

 

SetAsDefaultReference (bool bDef )

Makes a reference block the project's default reference block.

 

SetClonedBlock ( name blockname )

Sets the cloned block for a clone block.

 

SetInheritParameterValues ( bool inherit )

This options only applies for clone blocks and defines if parameters are inherited from cloned block. If set to false individual parameters for the clone block can be defined.

 

SetDeembedded ( bool deembed )

Sets the de-embedding property for a block, i.e. the inverse S-matrix will be considered for that block then.

 

SetDefineProbes( bool bFlag )

Activates probes on block ports for blocks that define probes.

 

SetCacheFiles( bool bFlag )

Enables/disables file caching for file reference blocks. Works only for blocks of type Touchstone file block.

 

SetSimulationTask( string taskfullname)

Sets the referenced simulation task. This is only supported for simulation project reference blocks (task must be a project simulation task) and hybrid solver reference blocks (task must be a hybrid solver task).

Iteration

StartBlockNameIteration int

Resets the iterator for the blocks and returns the number of blocks.

 

GetNextBlockName name

Returns the next block's name. Call StartBlockNameIteration before the first call of this method.

 

StartPropertyIteration

Resets the iterator for the properties of a block.

 

GetNextProperty ( string_ref propertyname, string_ref type, string_ref value )

Positioning

FlipHorizontal

Horizontally flips a block. If this setting is made before calling Create, the flip is only stored. It will then be done in Create (before any rotation is applied).

 

FlipVertical

Vertically flips a block. If this setting is made before calling Create, the flip is only stored. It will then be done in Create (before any rotation is applied).

 

GetPinPositionX ( int index ) int

Returns he horizontal position of a block pin specified by the given index. Index must be an integer between 0 and GetNumberOfPins()-1.

 

GetPinPositionY ( int index ) int

Returns he vertical position of a block pin specified by the given index. Index must be an integer between 0 and GetNumberOfPins()-1.

 

GetPositionX int

Returns the horizontal position of the center point of a block.

 

GetPositionY int

Returns the vertical position of the center point of a block.

 

Move ( string x, string y )

Moves a block by the given offset. Note that the schematic size is given by 0 < x, y <100000. Use Position to specify a certain location. It is always ensured that the block is aligned with the grid, therefore the specified offset might get adjusted slightly. This setting must not be made before calling Create.

 

Position ( string x, string y )

Specifies the position of the block. This setting must be made before calling Create. Furthermore, it can be used to modify a block's position. Possible values are 0 < x, y <100000, i.e. (50000, 50000) is the center of the schematic. It is always ensured that the block is aligned with the grid, therefore the specified position might get adjusted slightly.

 

Rotate ( string angle )

Rotates the block by the given angle in degrees around its center point. If this setting is made before calling Create, the angle is only stored. The rotation will then be done in Create (after any horizontal or vertical flip was applied).

Special Methods

ConvertToReferenceFileBlock ( string array paths, bool relative )

For some file blocks only.
Converts a project file block into a reference file block. For that, the file(s) used by the block are be copied to the given paths and then referenced. The file(s) inside the project which were used by the original block will be deleted after conversion.

 

ConvertToProjectFileBlock

For some reference blocks only.
Converts a reference file block into a project file block. A copy of the referenced files is moved into the block and the referenced files remain unchanged.

 

ImportLibraryModel (filename filename, string path ) string

Imports a new model form a spice or touchstone file into the device model library. The imported model must be compatible with the current block. The path parameter can be used to import the device model to a specific path in the device model library, pass an empty string if the model should be imported into the device base path. If a library model with the same name and path is already present in the device model library it will be overwritten. Use the command LibraryModelExists to check the existence of a model before import. The return value is the tree path in the device model library which can be used together with the SetLibraryModel command.

 

LibraryModelExists ( name model ) bool

Checks if a model exists in device model library. The name is the tree path in the device model library.

 

UpdateModel bool

Updates the model of block. Returns True if the block was updated.
Note: Currently used only for reloading IBIS-models.

 

GetSchematic variant

Opens the schematic sheet of an Array Block and returns a dispatch to it. Further VBA-commands can be executed in the corresponding document using the returned dispatch.

Dim ARRAY1DISPATCH As Variant

With Block

.Reset

.Name ("ARRAY1")

.Type ("ARRAY")

.Create

Set ARRAY1DISPATCH = .GetSchematic

With ARRAY1DISPATCH.ExternalPort

.Reset

.Name "1"

.Create

End With

End With

 

Close

Closes the corresponding schematic sheet after saving. Once closed, any dispatch obtained using GetSchematic becomes obsolete.

IBIS

ClearIbisData

Deletes the block's volatile data to release memory as described in the IBIS data storage policy. This command works with IBIS reference blocks only. Note that there is also a command to delete the volatile data of all IBIS blocks in the project.

 

GetIbisComponentName string

Returns the name of the selected IBIS component.

 

GetIbisPackageName string

Returns the name of the selected IBIS package.

 

GetIbisPinName( int pin_type ) string

Returns the name of the selected IBIS pin. pin_type may be 0 or 1 to refer to the plain or the inverting, respectively.

 

GetIbisBufferName( int pin_type, int index ) string

Returns the name of the selected IBIS buffer. pin_type may be 0 or 1 to refer to the plain or the inverting, respectively. index is an identifier, as returned by GetIbisBufferIndices.

 

GetIbisBufferIndices string[]

Returns a list of the indices associated with the block's selected buffers. These indices serve as identifiers for parameter sweeps and the GetIbisBufferName command.

 

GetIbisBufferParam string

Returns the parameter string.

 

GetIbisBufferParamActive bool

Tells whether the buffer parametrization is active.

 

GetIbisCorner int

Returns the value of the selected corner (0 for typical, 1 for fast, 2 for slow).

 

GetIbisCornerParam string

Returns the corner parameter string.

 

GetIbisCornerParamActive bool

Tells whether the corner parametrization is active.

 

GetIbisInternalPower bool

Tells whether the IBIS block is powered internally.

 

GetDefineProbes bool

Tells whether the block probes are enabled.

 

SetIbisComponent( string component ) bool

Sets the component of an IBIS reference block. The argument must match a name listed under the [Component] keyword in the block's IBIS file.

Returns True if no error occurred.

 

SetIbisPackage( string packageType, string packageName ) bool

Sets the package model of an IBIS reference block. The first argument must be one of 'None', 'Global', 'Pinwise', or 'Named'. If the latter is used, then a package model with the name given as second argument is sought. Otherwise the second argument is ignored.

Returns True if no error occurred.

 

SetIbisBuffer( string pin, string invPin, string[] models, string[] invModels, string modelParameter ) bool

Sets the properties of an IBIS buffer block. The arguments are pin name, name of inverting pin (for differential buffers), a list of model names, a list of model names for the inverting pin and a parameter name for parameterized models.

The inverting pin name and the model parameter name may be empty strings. The pin name, inverting pin name (unless empty) and model names must match pin and model names, respectively, defined for the chosen IBIS component. Only pin-buffer combinations that are specified in the IBIS file are permitted. Differential buffers can only be set if a valid pair of differential pins is given.

The model parameter can be used to select models from the given model list in simulation control tasks like parameter sweeps.

Returns True if no error occurred.

An example for the use of this and other IBIS specific commands is provided in the examples section.

 

SetIbisCorner( int index,  string cornerParameter ) bool

Sets the corner settings of an IBIS buffer block. The index can be 0 (typical), 1 (fast) or 2 (slow). The parameter can be used to parameterize the corner setting. If a non-empty string is passed, the parameter overrides the index.

Returns True if no error occurred.

 

SetIbisSupplyMode( string mode )

Sets the IBIS buffer block's power supply and ground reference mode. Valid modes are internal, external, and merged. In the internal mode, ideal references for the pull-up, pull-down, power clamp and ground clamp elements with the appropriate voltages are incorporated into the buffer model. Contrarily, in the external mode, the buffer block is endowed with four pins to connect external devices. The merged mode differs from external in that the pin for the pull-up element is merged with the power clamp's pin, and the pull-down element's pin is merged with the ground clamp's pin. The description of the IBIS buffer block provides a more detailed explanation.

 

GetIbisSupplyMode string

Tells the IBIS buffer bock's supply mode. Possible modes are internal, external, and merged. See SetIbisSupplyMode for an explanation of the modes.

 

EnableAutomaticProbeNaming( bool enable )

         Activates or deactivates the automatic naming of the internal probe of the IBIS buffer block.  

 

IsAutomaticProbeNamingEnabled bool

Tells whether the automatic naming of the IBIS buffer block is enabled.

 

SetProbeName( string probeName )

Sets the name of the internal probe of the IBIS buffer block. This command will only work if the automatic probe naming of the IBIS block buffers is disabled.  

 

GetProbeName string

Returns the name of the internal probe of the IBIS buffer block.

 

SetIbisEnableWindowing( bool enableWindowing )

Enables or disables the truncation of waveforms.

 

GetIbisEnableWindowing bool

Tells whether waveform windowing is enabled.

 

SetIbisExplWindowStart( bool explicit )

The argument determines whether the start time of a window is specified explicitly (through SetIbisWindowStart) or implicitly (through SetIbisHeadTolerance).

 

GetIbisExplWindowStart bool

Tells whether a window's start is specified explicitly.

 

SetIbisExplWindowEnd( bool explicit )

The argument determines whether the end time of a window is specified explicitly (through SetIbisWindowEnd) or implicitly (through SetIbisTailTolerance).

 

GetIbisExplWindowEnd bool

Tells whether a window's end is specified explicitly.

 

SetIbisWindowStart( string start )

Sets an expression that is evaluated to determine the start time of a window if an explicit start time is selected (through SetIbisExplWindowStart). The unit is assumed to be seconds.

 

GetIbisWindowStart string

Returns the expression specifying a window's start if an explicit start time is chosen.

 

SetIbisWindowEnd( string end )

Sets an expression that is evaluated to determine the end time of a window if an explicit end time is selected (through SetIbisExplWindowEnd). The unit is assumed to be seconds.

 

GetIbisWindowEnd string

Returns the expression specifying a window's end if an explicit end time is chosen.

 

GetIbisWindowLossStart double

Returns the relative loss of information at the begin of a waveform due to windowing. It is defined as the ratio of the absolute difference between the signal's initial value and the value at the window's start with to the signal's amplitude. It is equal to the head tolerance if the window's start is determined implicitly.

 

GetIbisWindowLossEnd double

Returns the relative loss of information at the end of a waveform due to windowing. It is defined as the ratio of the absolute difference between the signal's final value and the value at the window's end with to the signal's amplitude. It is equal to the tail tolerance if the window's end is determined implicitly.

 

SetIbisHeadTolerance( string headTolerance )

Sets a threshold for the variation of the waveforms's head. Variations less than the tolerance will be ignored in simulations (if windowing is enabled). The expression must evaluate to a value in the range from 0 to 0.1, or an error is raised.

 

GetIbisHeadTolerance string

Returns the expression specifying the tolerance for waveform windowing at the waveforms head.

 

GetIbisHeadTruncationTime( string waveform ) double

Tells the start time of a waveform window in seconds. It is determined implicitly through the head tolerance (set with SetIbisHeadTolerance) or explicitly through the start time (set with SetIbisWindowStart), depending on the explicit start setting (set with SetIbisExplWindowStart).

 

SetIbisTailTolerance( string tailTolerance )

Sets a threshold for the variation of the waveforms's tail. Variations less than the tolerance will be ignored in simulations (if windowing is enabled). The expression must evaluate to a value in the range from 0 to 0.1, or an error is raised.

 

GetIbisTailTolerance string

Returns the expression specifying the tolerance for waveform windowing at the waveform's tail.

 

GetIbisTailTruncationTime (string waveform)  double

Tells the end time of a waveform window in seconds. It is determined implicitly through the tail tolerance (set with SetIbisTailTolerance) or explicitly through the end time (set with SetIbisWindowEnd), depending on the explicit end setting (set with SetIbisExplWindowEnd).

 

SetIbisCustomTrigger( bool customize )

Enables or disables the custom trigger thresholds.

 

SetIbisHighThreshold( string threshold )

Sets the higher trigger threshold. This setting takes effect only if custom trigger thresholds are enabled.

 

SetIbisLowThreshold( string threshold )

Sets the lower trigger threshold. This setting takes effect only if custom trigger thresholds are enabled.

 

GetIbisCustomTrigger bool

Tells whether custom trigger thresholds are enabled. If custom trigger thresholds are disabled, the default values are used. The default value for both the lower and the higher threshold is half the reference voltage.

 

GetIbisHighThreshold string

Returns the expression set for the higher trigger threshold. This setting takes effect only if custom trigger thresholds are enabled.

 

GetIbisLowThreshold string

Returns the expression set for the lower trigger threshold. This setting takes effect only if custom trigger thresholds are enabled.

Interpolation

CanSelectInterpolationMethod bool

Checks if the user can select between different interpolation methods for the given block. Currently, this is only possible for Touchstone blocks. Hence, the return value indicates, whether the set-methods SetUseHigherOrderInterpolation(), SetHigherOrderInterpolationAccuracy() and the get-methods GetUseHigherOrderInterpolation(), GetHigherOrderInterpolationAccuracy() can be called.

 

CopyInterpolationDataToProject

Sets the block's interpolation data for all blocks of the same type that are contained in your model. This is only possible for some types of blocks.

 

GetUseHigherOrderInterpolation bool

The return value indicates whether higher-order interpolation is used for the S-Parameter data of the block or linear interpolation. The method can only be applied to blocks that able to switch between different interpolation methods. This can be queried by the method CanSelectInterpolationMethod().

 

GetHigherOrderInterpolationAccuracy string

Returns the desired accuracy applied on a rational fit for higher-order interpolation of the S-Parameter data of the block. The method can only be applied to blocks that able to switch between different interpolation methods. This can be queried by the method CanSelectInterpolationMethod().

 

SetInterpolation ( bool interpolate )

Switches on/off interpolation for a block. This is only possible for some types of blocks.

 

SetInterpolationDataForProperty ( name propertyname, string min, string max, string samples )

Sets the interval for a block's property inside that interpolation will be performed with a given number of samples. This is only possible for some types of blocks.

 

SetUseHigherOrderInterpolation( bool bFlag )

The flag specifies whether higher-order interpolation is to be used for the S-Parameter data of the block or linear interpolation. The method can only be applied to blocks that able to switch between different interpolation methods. This can be queried by the method CanSelectInterpolationMethod().

 

SetHigherOrderInterpolationAccuracy( string accuracy )

Sets the desired accuracy applied on a rational fit for higher-order interpolation of the S-Parameter data of the block. The method can only be applied to blocks that able to switch between different interpolation methods. This can be queried by the method CanSelectInterpolationMethod().

Returns the bus size of the block pin specified by pinindex. Bus size is the number of electrical connections that the block pin carries. Non bus block pins have the bus size 1.

Returns the block ports inside a bus pin. true is returned if the operation was successful.
(See Example)

 

GetNumberOfNonReferencePorts int

Returns a block's number of ports (without reference ports).

Returns the block's number of pins (including reference pins).

 

GetNumberOfPorts int

Returns the block's number of ports (including reference ports).

 

GetPinLayout ( int pinindex, string_ref edge, int_ref edgeindex  )

Queries how the layout of the block pin with the specified pin index is defined. Writes one of the edges listed for SetPinLayout() into the variable edge and the index of the pin on that edge into the variable edgeindex. Edge indexing is from top to bottom and from left to right.

(See Example)

 

GetPinLayoutType enum

Queries how the pin layout of a block is defined. "fixed" is returned for blocks that do not support different pin layout settings. For all other blocks one of the types listed for SetPinLayoutType() is returned.

 

GetDifferentialCablePorts bool

Queries whether differential ports for a block using cable models are set.

 

GetDifferentialPorts bool

Queries whether differential ports are set for a block.

 

GetPinName ( int pinindex ) string

Returns the name of a block pin specified by the given index. Index must be an integer between 0 and GetNumberOfPins()-1.

 

SetPinName ( int pinindex, name pinname )

Set the name of a block pin specified by the given index. Index must be an integer between 0 and GetNumberOfPins()-1. Specifying an empty name resets the pin name to the default name.

Note: call ApplyPinLayout to apply changes made by this command

(See Example)

 

GetPinIndex ( name pinname ) int

Returns the index of a block pin specified by the given pinname. If a pin was found the returned index is an integer between 0 and GetNumberOfPins()-1. The function throws an error and returns -1 if no pin was found.

 

GetPortIndex ( name portname ) int

Returns the index of a block port specified by the given portname. If a port was found the returned index is an integer between 0 and GetNumberOfPorts()-1. The function throws an error and returns -1 if no port was found.

(See Example)

 

GetPortLabel ( int portindex ) string

Returns the label of a block port specified by the given portindex. Since not all blocks support port labels, this function may return an empty string. Index must be an integer between 0 and GetNumberOfPorts()-1.

(See Example)

 

GetPortName ( int portindex ) string

Returns the name of a block port specified by the given portindex that must be an integer between 0 and GetNumberOfPorts()-1.

(See Example)

 

IsPinConnected( int pinindex) bool

Checks if the specified pin of this block is connected. pinindex must be an integer between 0 and GetNumberOfPins()-1.

 

ApplyPinLayout() bool

To improve performance manual layout setter commands like SetPinName and SetPinLayout are saved in the background until this command is called to apply them.

Automatic layout setter commands like SetPinLayoutType and SetDifferentialPorts change the block directly and don't need to call this function afterwards.

(See Example)

 

SetPinLayout ( int pinindex, string edge, int edgeindex  )

Sets the location of a block pin specified by pinindex (i.e. 0,LEFT,1 => moves the pin with index 0 to be the second pin on left). This is only possible for some types of blocks. 0 based edge indexing is from top to bottom and from left to right. The following edges can be set:

"BOTTOM"

Bottom edge.

"LEFT"

Left edge.

"RIGHT"

Right edge.

"TOP"

Top edge.

Note: call ApplyPinLayout to apply changes made by this command.

(See Example)

 

SetPinLayoutType ( enum type )

Sets how the pin layout of a block is defined, i.e. the position of each pin. This is only possible for some types of blocks. The following types are supported:

"manual"

The pins are manually positioned.

"left right clockwise"

The pins are distributed on the left and right edges in clockwise order.

"left right anticlockwise"

The pins are distributed on the left and right edges in anticlockwise order.

"left right alternating"

The pins are distributed on the left and right edges in alternating order.

"left right downward"

The first half of the pins is located on the left edge from top to bottom, and the second half of the pins is located on the right edge from top to bottom.

"clockwise"

The pins are distributed on all four edges in clockwise order.

"anticlockwise"

The pins are distributed on all four edges in anticlockwise order.

"automatic"

The pins are distributed for optimal connector routing.

Note: this command overwrites changes made by SetPinName and SetPinLayout.

(See Example)

 

SetDifferentialCablePorts ( bool differential )

Sets differential ports for a block using cable models, i.e. its reference pins can also be accessed. This is only possible for SimLab TL/PEEC file blocks and CST Cable Studio blocks.

Note: this command overwrites changes made by SetPinName and SetPinLayout.

 

SetDifferentialPorts ( bool differential )

Sets differential ports for a block, i.e. its reference pins can also be accessed. This is only possible for some types of blocks.

Note: this command overwrites changes made by SetPinName and SetPinLayout.

and the set-methods:
  SetConvolutionFmax, SetMOR_AccuracyAbs, SetMOR_InitialPoles, SetMOR_MaximumPoles, SetMOR_PassivateToInfinity, SetMOR_PoleIncrement, SetUseNaturalImpedances, SetMOR_RenormalizationImpedance, SetVectorFittingMethod, can be called.

 

DeleteMORCache

The MOR cache caches the poles and residues of a reduced order model for efficiency reasons since the Vector Fitting of a block can sometimes be costly and should not be repeated in every transient simulation. By this method, all cache entries for the block can be deleted. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

StartIdEM( string fmax , string task )

 Starts our specialized tool for macro model generation IdEM in an interactive mode with the frequency domain representation of the block used as input. The interactive mode is such that all relevant macro modelling techniques can be applied to the input data and generated macro models can be exported to Design Studio. Doing this, macro modelling results and the applied vector fitting settings of the block are stored internally in the Design Studio project.

The arguments of the method specify the frequency range over which the data is fitted:
If both arguments are empty strings, the block frequency limits are applied, provided the block has frequency limits.
If fmax is an empty string, the frequency range of the circuit simulation task task is applied.
If task is an empty string, the frequency range between DC and fmax in project frequency units is applied.

The method can only be applied to blocks that are represented by a single N-Port matrix and have native port impedances. This is a subset of all blocks, employing Vector Fitting, and does not include Cable Studio and PCB Studio blocks.

 

ExtractStateSpaceModel( string fmax , string task )

  Performs a Vector Fitting on the block, based on the frequency range defined by the arguments of the method, and calculates a state space representation from vector fitting results. The ABCD state space matrices are then exported in csv format together with a logfile that describes the format of the state space representation together with relevant meta data. The files are exported into the "ABCD" subfolder of the general "Export" folder of the project. The naming convention is such that all exported files carry the name of the block and have a suitable extension. In addition ABCD matrices carry a postfix "_A" to "_D", respectively.

The arguments of the method specify the frequency range over which the data is fitted:
If both arguments are empty strings, the block frequency limits are applied, provided the block has frequency limits.
If fmax is an empty string, the frequency range of the transient-, AC-, or S-Parameter task task is applied.
If task is an empty string, the frequency range between DC and fmax in project frequency units is applied.

The method can only be applied to blocks that are represented by a single N-Port matrix and have native port impedances. This is a subset of all frequency-dependent blocks and does not include Cable Studio and PCB Studio blocks.

GetVectorFittingMethod string

SetMOR_AccuracyAbs( string value )

value specifies the desired approximation accuracy of S-Parameters in Vector Fitting. The accuracy is specified as an absolute value. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

SetMOR_EnhanceAccuracyAtDC ( bool enhance )

By this method, the vector Fitting algorithm can be induced to put more emphasis on the model accuracy at DC to improve the steady-state behavior of the model. But note that the overall accuracy of the model might be decreased when demanding high accuracy at DC. Note also that it might not be possible to achieve a high accuracy at DC if there are strong passivity violations of the input data at DC. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

SetMOR_ErrorNorm( string method )

Specifies the kind of error norm applied by the Vector Fitting algorithm. The argument can assume the values "Max" or "L2". The maximum norm considers the maximum error within the frequency range of interest while the L2 norm averages the error over the frequency range. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

SetMOR_InitialPoles( string poles )

Specifies the initial number of poles used in Vector Fitting. If the argument is zero by value, the required number of poles is estimated automatically in the MOR algorithm. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

SetMOR_ManualPoleChoice ( bool choice )

The argument choice indicates whether the initial number of poles and/or the pole increment, used in Vector Fitting, are derived from manual user settings or calculated automatically, disregarding settings by the user, during transient simulation. If choice is true, the initial number of poles and the pole increment can subsequently be set via the methods SetMOR_InitialPoles and SetMOR_PoleIncrement, respectively. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

 SetMOR_MaximumPoles( string poles )

Specifies the maximum number of poles used in Vector Fitting. If the argument is zero by value, the required number of poles is estimated automatically in the Vector Fitting algorithm. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

SetMOR_PassivateToInfinity( bool passivatetoinfinity )

Specifies whether S-Parameters are to be passivated to an infinite frequency in Vector Fitting. This guaranties absolute stability of the reduced order model, but it might reduce the accuracy of the approximated S-Parameters in the frequency range of interest. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

SetMOR_PoleIncrement( string increment )

Specifies the increment in the number of poles used in Vector Fitting. In the algorithm the number of poles is increased by the increment from iteration to iteration, starting at the initial number of poles. If the argument is zero by value, an appropriate pole increment is calculated automatically by the Vector Fitting algorithm. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

SetUseNaturalImpedances( bool bFlag )

The flag specifies whether natural port impedances are used as reference for the convolution of a Vector Fitting model, used in transient simulation or, if false, whether a fixed impedance is used as reference impedance for all ports. Natural port impedances are advantageous, because they tend to lead to shorter impulse responses and to a lower number of significant poles. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR().

 

SetMOR_RenormalizationImpedance( string resistance )

Specifies a positive resistance to which all ports impedances are renormalized when S-Parameters are supplied to the Vector Fitting. The argument is in Ohms. Note that the renormalization is only carried out if no natural port impedances are used. This managed by the method SetUseNaturalImpedances. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

 

SetConvolutionFmax( string fmax )

Specifies the maximum frequency to which S-Parameters are fitted in Vector Fitting. The argument is in frequency units. If the argument is zero by value, the maximum frequency is calculated automatically, based on the transient excitation. The method can only be applied to blocks that can use convolution via MOR or IFT. This can be queried by the method CanUseMOR.

 

SetVectorFittingMethod( string method )

Specifies the kind of method applied for Vector Fitting. The argument can assume the values "BUILT-IN" or "IDEM" where BUILT-IN is our legacy pole relocation vector fitting method and IDEM is the vector fitting technique used by our specialized tool for macromodel generation. The method can only be applied to blocks that use Vector Fitting to represent dispersive data. This can be queried by the method CanUseMOR.

'Change the layout of the block

'Precondition: block 'MyBlock' have at least 2 pins on the left and 2 pins on the right

Dim edge As String

Dim edgeindex As Long

With Block

   .Reset

   .Name "MyBlock"

 

   .SetPinLayoutType( "clockwise" ) 'The pins are distributed on all four edges in clockwise order.

   'Layout is automatically applied.

 

   .SetDifferentialPorts(True) 'Sets differential ports For a Block.

   'Layout is automatically applied.

 

   .SetPinName( 0, "pin1" ) ' sets the name of pin 0

   .SetPinName( 1, "pin2" ) ' sets the name of pin 1

   .ApplyPinLayout() 'Apply layout changes that were made in the background.

 

   .GetPinLayout( 0, edge, edgeindex ) ' gets the location of pin 0

 

   .SetPinLayout( 0, edge, edgeindex+1 ) ' moves the pin 0 down on the same edge

   .ApplyPinLayout() 'Apply layout changes that were made in the background.

 

   .SetPinLayout( 0, "TOP", 0 ) ' moves the pin 0 to be the first pin on the top.

   .SetPinLayout( 1, "LEFT", 0 ) ' moves the pin 0 to be the first pin on the left.

   .ApplyPinLayout() ' Apply layout changes that were made in the background.

End With
