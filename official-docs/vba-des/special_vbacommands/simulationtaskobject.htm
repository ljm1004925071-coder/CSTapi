General Methods

Reset

Resets all internal settings to their default values.

 

Type ( string type )

Sets the type for a simulation task. This setting must be made before calling Create. The type can not be modified after the creation. Most task settings can be accessed by a simulation task object. In addition, specialized objects are available for some types of tasks. Valid task types and specialized objects are:

Name ( name taskname )

Sets the name of a simulation task before calling Create. Furthermore, this method can be used to select an existing task of your project. The task name needs to include all parent tasks for nested tasks, separated by backslashes, e.g. "Seq1\SPara1".

 

Create

Creates a new simulation task.

 

Delete

Deletes the task with the given name. Name needs to be called before this function can be evaluated.

 

Duplicate( bool copyResults ) string

Duplicates the task with the given name. copyResults specifies if available results are copied in the duplicated task. The name of the duplicated task is returned.

 

Update

Executes the task with the given name, including all sub tasks. Name needs to be called before this function can be evaluated.

 

Note: To update all Tasks in the tree, the command  "UpdateResults" can be used. This command is not part of the object SimulationTask.

 

SetName ( name taskname )

Modifies the name of an existing simulation task. The name needs to include parent tasks, like for Name. By specifying a different parent part it is also possible to move a task from one parent to another.

 

MoveInTree ( name newparent, name nexttask )

Moves an existing simulation task to a different position in the tree. newparent specifies the new parent task (including parents of the parent like for Name). If newparent is empty the task will be moved to the top level. If nexttask is empty the task will be moved behind all tasks at the new level. If nexttask is not empty the task will be moved in front of nexttask. nexttask specifies the task name only (without parent tasks).

 

ValidateSetup

Verifies that all task settings respect their restrictions. The function either writes a confirmation to the message window, or raises an error with a message explaining the problems found.

Methods Concerning Properties

SetProperty ( name propertyname, string value )

 

Sets the given value for the specified property. The following properties are available for the different simulation task types:

All tasks

"enabled" (possible values: "True" or "False"): enables/disables a simulation task.

All circuit simulation tasks

"parallelization " (possible values: "Maximum", "None" or any positive integer): specifies how much parallelization is applied, the maximum possible number of threads, no parallelization, or a custom number of threads, respectively.

"auto solver" (possible values: "True" or "False"): specifies whether the Nonlinear solver settings in the Solver Specials dialog of the task are calculated automatically and manual settings are ignored.

"error current": specifies tolerance on error current (=sum of all currents to/from a node) in simulation

"tolerance type" (possible values: "relative" or "absolute"): Type of tolerance on error current. Reference value is the impressed current.

"iterations": specifies the maximum number of iterations in the calculation of the initial operation point.

"auto condition" (possible values: "True" or "False"): specifies whether the Circuit condition settings in the Solver Specials dialog of the task are calculated automatically and manual settings are ignored.

"gmin": conductance to ground, attached to possibly floating nodes to achieve convergence in circuit simulation. The conductance is calculated automatically if the expression evaluates to a negative number.

"rmin": resistance value, below which resistors are treated as ideal shorts in circuit simulation.

"s-parameter interpolation scheme"  (possible values: "real/imaginary" "magnitude/phase"): specifies the way how S-parameter data can be interpolated.

S-Parameters

"fmin", "fmax": simulation frequency range. These settings are only used if "maximum frequency range" is off.

"maximum frequency range" (possible values: "True" or "False"): enables or disables the maximum frequency sweep range. If the maximum frequency range is disabled, the frequencies set by the properties "fmin" and "fmax" will be used. If the maximum frequency range is enabled, those properties are ignored. The maximum frequency range must be disabled if the model is not frequency limited.

"td block representation" (possible values: "True" or "False"): indicates whether a time domain representation, i.e., vector fitting, is used for frequency-dependent blocks.

"circuit simulator": the type of simulator to be used (possible values: "cst" or "hspice"). For the latter an HSPICE netlist is exported. If the path to a Synopsis HSPICE executable has been specified via Macros->Solver->HSPICE Settings, an Synospsis HSPICE simulation on the exported netlist is carried out in addition.

"broadband sweep" (possible values: "True" or "False"): Specifies whether an accuracy-controlled, adaptive broadband frequency sweep is applied on task results of the S-Parameter task.

"broadband sweep accuracy": Specifies the desired accuracy of S-Parameters for the broadband frequency sweep applied on task results of the S-Parameter task. I.e., it defines the termination condition of the adaptive fitting process.

"block broadband sweep" (possible values: "True" or "False"): Specifies whether an accuracy-controlled, adaptive broadband frequency sweep is applied on block S-Parameters for blocks that are expensive to evaluate. The setting is ignored for blocks that are inexpensive to evaluate.

"block broadband sweep accuracy": Specifies the desired accuracy of block S-Parameters if a broadband frequency sweep is applied on the results of a block. I.e., it defines the termination condition of the adaptive fitting process.

Operating point (DC)

"circuit simulator": the type of simulator to be used (possible values: "cst", "hspice", "pspice" or "ltspice"). For the latter three a SPICE netlist is exported. If the path to a Synopsis HSPICE or LTSPICE executable, respectively, has been specified, a SPICE simulation on the exported netlist is carried out in addition.

"vector fitting" (possible values: "True" or "False"): indicates whether a time domain representation, i.e., vector fitting, is used for frequency-dependent blocks. Use this option if you want the operation point task results to be consistent with the initial conditions of a transient task or the task results of a Frequency Domain task that uses vector fitting. Note that all circuit tasks calculate an operation point at the beginning of the actual simulation and that in the operation point calculation vector fitting results of frequency-dependent blocks are applied if they are available.

"vector fitting fromtask" (possible values: "True" or "False"): If true and if vector fitting is enabled, the frequency range of another circuit task is inherited for the vector fitting of the operation point task. If false and if vector fitting is enabled, the frequency range in vector fitting is expected to be specified manually.

"vector fitting task": If the property "vector fitting fromtask" is enabled, this property specifies the name of a circuit task, from which the frequency range, applied on vector fitting, is inherited. The operation point task results are thus made consistent with the operation point calculation of that circuit task.

"vector fitting fmax": If vector fitting is enabled and if the property "vector fitting fromtask" is disabled, this specifies the upper value of the frequency range, applied in vector fitting. DC is applied as lower frequency limit as long as the respective block supports DC.

"block broadband sweep" (possible values: "True" or "False"): Specifies whether an accuracy-controlled, adaptive broadband frequency sweep is applied on block S-Parameters for blocks that are expensive to evaluate. The setting is ignored for blocks that are inexpensive to evaluate and if vector fitting is disabled for the operation point task.

"block broadband sweep accuracy": Specifies the desired accuracy of block S-Parameters if a broadband frequency sweep is applied on the results of a block. I.e., it defines the termination condition of the adaptive fitting process.

Transient

"integration method" (possible values: "adams" or "gear")

"integration order" (possible values: 1, 2, 3, or 4)

"time stepping" (possible values: "automatic" ,"adaptive", and "fixed")

"time step": maximum/fixed time step for adaptive/fixed time stepping.

"tmax": to be simulated time span.

"minimum time step": minimum time step for adaptive time stepping

"fmin", "fmax": Reference frequency range. They are set per default to 0 and 1.

"calculatefullsparameter": (possible values: "True" or "False"). Enables/disables the full S-Matrix calculation.

"transient accuracy": the desired relative accuracy of the transient solution.

"circuit simulator": the type of simulator to be used (possible values: "cst", "cst - steady state", "cosimulation", "hspice", "pspice" or "ltspice"). For the latter three a SPICE netlist is exported. If the path to a Synopsis HSPICE or LTSPICE executable, respectively, has been specified, a SPICE simulation on the exported netlist is carried out in addition..

"sampling method" (possible values: "Automatic", "Nyquist", "Manual", and "Simulated"). Note that the method SetSweepData() resets the sampling method to "Manual".

"fmax estimator": the type of estimation on the maximum excitation frequency (possible values: "automatic", "energy", "transitiontime", or "manual fmax").

"energy threshold": relative energy threshold for the estimation of the maximum excitation frequency. It only applies if the energy-based fmax estimator has been selected.

"manual fmax": maximum excitation frequency. It only applies if the manual fmax estimator has been selected.

"nfdsamples": The number of frequency sample points used for calculating spectral densities or FD voltages/currents.

"docombineresults": Starts the field combination in the 3D solver with respect to the calculated excitations in DS. Be sure to set the property "blocknameforcombineresults" to activate combine results correctly.

"blocknameforcombineresults": Selects the MWS - block for which the fields are to be combined.

"block broadband sweep" (possible values: "True" or "False"): Specifies whether an accuracy-controlled, adaptive broadband frequency sweep is applied on block S-Parameters for blocks that are expensive to evaluate. The setting is ignored for blocks that are inexpensive to evaluate.

"block broadband sweep accuracy": Specifies the desired accuracy of block S-Parameters if a broadband frequency sweep is applied on the results of a block. I.e., it defines the termination condition of the adaptive fitting process.

"steady state period detection" (possible values: "true" or "false"): Applicable if "cst - steady state" is enabled as " circuit simulator". In this case, it defines whether a suitable steady state period is extracted from periodic excitations at external ports or whether a manually specified steady state period is applied is steady state simulation.

"steady state period". (possible values: any time period to which all periodic excitations are commensurable. If they are not commensurable this will result in a simulation error.): Applicable if "cst - steady state" is enabled as " circuit simulator " and steady state period detection has been disabled. In this case, it specifies the steady state period in time units that is applied is steady state simulation.

"steady state tolerance" (possible values: between 0 and 1): Applicable if "cst - steady state" is enabled as "circuit simulator". In this case, it specifies a threshold on the relative resuidual of steady state iteration, below which steady state iterations are terminated and steady state simulation is considered successful.

AC

"fmin", "fmax": simulation frequency range. These settings are only used if "maximum frequency range" is off.

"maximum frequency range" (possible values: "True" or "False"): enables or disables the maximum frequency sweep range. If the maximum frequency range is disabled, the frequencies set by the properties "fmin" and "fmax" will be used. If the maximum frequency range is enabled, those properties are ignored. The maximum frequency range must be disabled if the model is not frequency limited.

"docombineresults": Starts the field combination in the 3D solver with respect to the calculated excitations in DS. Be sure to set the property "blocknameforcombineresults" to activate combine results correctly.

"blocknameforcombineresults": Selects the MWS - block for which the fields are to be combined.

"td block representation" (possible values: "True" or "False"): indicates whether a time domain representation, i.e., vector fitting, is used for frequency-dependent blocks.

"circuit simulator": the type of simulator to be used (possible values: "cst", "hspice", "pspice" or "ltspice"). For the latter three a SPICE netlist is exported. If the path to a Synopsis HSPICE or LTSPICE executable, respectively, has been specified, a SPICE simulation on the exported netlist is carried out in addition.

"block broadband sweep" (possible values: "True" or "False"): Specifies whether an accuracy-controlled, adaptive broadband frequency sweep is applied on block S-Parameters for blocks that are expensive to evaluate. The setting is ignored for blocks that are inexpensive to evaluate.

"block broadband sweep accuracy": Specifies the desired accuracy of block S-Parameters if a broadband frequency sweep is applied on the results of a block. I.e., it defines the termination condition of the adaptive fitting process.

Methods Concerning Result Options

EnableResult ( string resultname, bool enable )

Switches on/off the consideration of the result specified by name. The following results are available:

S-Parameters

"block"

Operating point (DC)

"block"

Transient

"block"

AC

"block"

Spectral lines

none

Mixer

"vswr", "spectrum", "isolation", "gain"

Amplifier

"s-param", "group delay", "linear gain", "stability", "spectrum", "distortion", "power", "gain"

The result name "block" is quite generic and requires further explanation. Many blocks are S-Parameter based. Their block S-Parameters may be of interest and therefore they have an option "Create S-parameter" in their Solver Properties. Furthermore, for some circuit tasks S-Parameter based blocks are characterized by vector fitting. The results of this vector fitting process might also be of interest. Enabling the "block" result option, block S-Parameters and/or vector fitting results are displayed in the tree for all S-Parameter based blocks, that have the "Create S-parameter" block solver option enabled, when running the task.

 

ResultEnabled ( string resultname ) bool

Tells whether the result specified by name is considered. The available result names are identical to those used by EnableResult.

 

GetResultOption ( string resultoption_name ) string

Returns the value of the named result option of  the selected task.

resultoption_name:
These values are case sensitive and are the same as displayed in the Name column in the Results Settings tab in the Simulation Task dialog.  

An error is thrown if no or a non existing task is specified or if the given name couldn't be recognized as a result option of the task .

 

SetResultOption ( string resultoption_name , string resultoption_value  )

Sets the value of the named result option of  the selected task to the named value.

resultoption_name:
Specifies the result option to be modified.

These values are case sensitive and need to be the same as displayed in the Name column in the Results Settings tab in the Simulation Task dialog.  
An error is thrown if the name couldn't be recognized as a result option name of the task. An error is thrown if no task or a non existing task is specified.

resultoption_value:
Is the value to which the specified  result option will be set. For enum type resultoption_values, the string is case sensitive and needs to agree to the available values in the corresponding task parameter window.
An error is thrown if the task doesn't accept the value (for instance, if a non existing value of a enum variable is given).  
Boolean resultoptions (displayed as checkbox in the block parameter window) are set by "true" or "false" strings. Both don't need to be case sensitive. Variants as "True" or "tRuE" are also accepted. Any other string will throw an error.

The following two VBA methods do only apply to result options that have an Auto checkbox in the Control column. Currently, only result options of transient tasks have a Control column. The mechanism of such auto checkboxes is as follows: If the checkbox is checked, an automatically calculated value is displayed in the non-editable edit field of the Expression column. If the checkbox is unchecked, a manual value can be edited in edit field of the Expression column.


GetResultOptionAutoControl ( string resultoptiondescription ) bool

Returns whether the "Auto" checkbox in the 'Control' column of the named result option of  the selected task is checked.

Parameter resultoptiondescription: (case sensitive as displayed in the 'Results Settings'-tab in the 'Simulation Task'-dialog). An error is thrown if the name couldn't be recognized as a result option name of the task or if the named result option does not have an "Auto" checkbox control.

 

SetResultOptionAutoControl ( string resultoptiondescription , bool resultoptionstate  )

Sets the state of the "Auto" checkbox in the 'Control' column of the named result option of  the selected task.

Parameter resultoptiondescription: (case sensitive as displayed in the 'Description'-column in the 'Results Settings'-tab in the 'Simulation Task'-dialog). An error is thrown if the name couldn't be recognized as a result option name of the task. An error is thrown if no task is selected or if the given name couldn't be recognized as a result option of the task or if the named result option does not have an "Auto" checkbox control.

Parameter resultoptionstate: . If true, the "Auto" checkbox in the 'Control' column is checked, otherwise it is unchecked. An error is thrown if the task couldn't set the value. Note that a Boolean result option (displayed as checkbox) can be set using the value "true" or case-insensitive variants such as "True" or "tRuE" but setting it to "1" would throw an error.

Special Methods

SetLogarithmicSweep( bool bLog )

Switches on/off the logarithmic sweep. An error is thrown if logarithmic sweep is not supported by the simulation task.

 

SetSweepType ( string sweeptype )

Specifies the property to sweep. Valid arguments are "no", "frequency" and "power" for amplifier tasks and "no", "frequency", "lo power" and "input power" for mixer tasks.

 

SetSweepData ( string min, string max, string samples )

Specifies the sweep range and the number of samples. The sweep range is Tmin and Tmax for transient tasks. For frequency domain tasks it is Fmin and Fmax, and for mixer and amplifier tasks it is either Fmin and Fmax or Pmin and Pmax (depending on the task settings). Note that for transient tasks the sampling method is reset to "Manual".

 

SetLocalUnit ( string unittype, string unit )

Sets the task to local units, i.e. values of all properties associated with the selected simulation task will not refer to the project's global units any more but to the tasks local units. Additionally, the unit of the specified type of unit is set to the given value.

 

SetGlobalUnit ( string unittype )

Sets the task to global units. The local unit of the specified type is deleted. All other local units are preserved, but not active. They can be activated again by calling SetLocalUnit.

 

SetDiamondScheme ( bool diamond )

Switches on/off the consideration of the frequencies according to the diamond scheme for a spectral lines simulation task.

 

SetDiamondHarmonics ( string harmonics )

Sets the number of harmonics for the diamond scheme for a spectral lines simulation task.

 

AddFundamentalFrequency ( string frequency, string harmonics )

Adds a fundamental frequency to a spectral lines simulation task with a given number of harmonics that will be considered if the diamond scheme is switched off.

 

ClearFundamentalFrequencies

Removes all entries from the list of fundamental frequencies for a spectral lines simulation task.

SetComplexPortExcitation (string portname, string amplitude,  string phase)

Sets the complex excitation amplitude of an AC-Task. The task needs to exist before this function can be called.

 

SetRealPortExcitation (string portname, string amplitude)

Sets the excitation amplitude of a DC-Task. The task needs to exist before this function can be called.

 

SetPortSourceType ( string portname, string type )

Sets the source type associated with the given port for the selected simulation task. Possible values are:

"None"

"Voltage"

"Current"

 "Signal"

 

The value "None" deletes the excitation signal of this port. The port will therefore be considered as impedance.

The task needs to exist and "SetComplexPortExcitation" ("SetRealPortExcitation") has to be called before this function can be executed.

 

SetPortSignalDelay ( string portname, string delay)

Sets the signal delay for an already existing transient excitation signal of port "portname".

An error will be thrown if the port could not be found or if no signal is assigned to it.

An error will also be thrown if the given task does not exist or if it is not a transient task.

 

SetPortSignalName ( string portname, string signalname)

Sets a user defined name for an already existing excitation signal of port no. portname.

 

SetPortImpedanceBlockDependent ( name portname )

Sets the impedance of the associated port for the selected simulation task to be taken from the connected block (if possible).

 

SetPortImpedanceFileName ( name portname, string filename )

Sets the filename associated with the given port for the selected simulation task. The impedance data will be imported from the given file. The setting will not be set if no data could be imported from the file.

 

SetConstPortImpedance ( name portname, string real, string imaginary )

Sets the impedance associated with the given port for the selected simulation task. Impedances are only available for ports which are not excited with current or voltage. This function is also used to set the reference impedance if the port excitation source type is "Signal". This function sets the constant property also if it is not used by the simulation.

 

SetPortInnerConductance ( name portname, string conductance )

Sets the inner conductance associated with the given port for the selected simulation task.

This setting is only used if the port is excited with "Current".

 

SetPortInnerResistance ( name portname, string resistance )

Sets the inner resistance associated with the given port for the selected simulation task.

This setting is only used if the port is excited with "Voltage".

 

SetUseExcitationReferenceFminFmax( string portname, string boolean)

Sets the corresponding port to use reference Fmin/Fmax of the corresponding task.

 

LoadLisFile

This method is only applicable for circuit simulation tasks that have "Run Synopsys HSPICE" specified as "Circuit Simulator" in their task parameter list. Furthermore, it requires that the Synopsys HSPICE simulator has been executed on the HSPICE netlist, generated by the circuit simulation task. A prerequite for this is that a valid path to a Synopsys HSPICE executable has been specified by the user. The method reads the results from the lis-file, generated by Synopsys HSPICE, and puts them into the Design Studio result tree.

 

LoadRawFile

This method is only applicable for circuit simulation tasks that have "Run LTSPICE" specified as "Circuit Simulator" in their task parameter list. Furthermore, it requires that the LTSPICE simulator has been executed on the LTSPICE netlist, generated by the circuit simulation task. A prerequite for this is that a valid path to an LTSPICE executable has been specified by the user. The method reads the results from the raw-file, generated by LTSPICE, and puts them into the Design Studio result tree. .

Queries

ArePortsDependent ( string portname1, string portname2 ) bool

Checks if two external ports specified by their names are dependent. The method returns True if the two ports are connected through some components of the model.

 

GetUnit ( string unittype ) string

Returns the unit for the given type specified for the selected task. This is either the task's local unit or the project's global unit.

 

GetUnitScale ( string unittype ) double

Returns the unit scale for the given type specified for the selected task. This is either the task's local scale or the project's global scale.

 

GetNameOfCurrentTask name

Returns the name of the currently running task while the simulation is performed.

 

GetModelFolder ( bool absolute ) string

Returns the generic model folder of the selected task, or an empty string if no task is selected. If absolute is false, the path is returned relative to the project model folder that can be obtained with GetProjectPath("ModelDS"). Note that not all tasks store individual model files, so the returned folder might not contain any files or not exist.

 

GetResultFolder ( bool absolute ) string

Returns the generic result folder of the selected task, or an empty string if no task is selected. If absolute is false, the path is returned relative to the project result folder that can be obtained with GetProjectPath("ResultDS"). Note that the returned folder is only used for result files without tree item. 1D results and tables are stored in different locations. You can get those paths via the ResultTree Object from the corresponding tree names.

Returns the impedance type associated with the given port for the selected simulation task.

The possible return types are:

"Import_File"

For DC-Task: only the real part is relevant (imaginary in this case is 0). This function returns always the saved constant property without any information whether it is used by the simulation.

Returns the conductance associated with the given port for the selected simulation task.

 

GetPortInnerResistance ( name portname ) string

Returns the resistance associated with the given port for the selected simulation task.

An error will be thrown if the port could not be found or if no signal is assigned to it.

An error will also be thrown if the given task does not exist or if it is not a transient task.

 

GetPortSourceType ( string portname ) string

Returns the source type associated with the given port for the selected simulation task. Possible return values are "None" (if no signal is assigned to the specified port, i.e. the port will be considered as impedance), "Voltage", "Current" and "Signal".

 

GetGaussProperties ( string portname, double& amplitude, double& fmin, double& fmax )

Retrieves the settings for a gaussian excitation signal at the given port.

 

StartTaskNameIteration int

Resets the iterator for the simulation tasks and returns the number of simulation tasks.

 

GetNextTaskName name

Returns the next simulation task's name. Call StartTaskNameIteration before the first call of this method.

 

DoesExist bool

Checks if a task with the given name does already exist.

 

GetTypeForTask ( name taskname ) name

Returns the type of the simulation task given by name.

 

GetSequenceTaskLoopCurrentIteration int

If a sequence task is currently performing a loop, this function returns the current iteration number. If no loop is currently running, -1 is returned.

 

GetSequenceTaskLoopMaxIteration int

If a sequence task is currently performing a loop, this function returns the maximum number of iterations. If no loop is currently running, -1 is returned.

 

GetUseExcitationReferenceFminFmax( string portname) bool

Returns true if the port is set to use reference Fmin/Fmax, otherwise false.

Dim arr() As String

With SimulationTask

    '1. Defining a sine signal

    .Reset

    .Name "Tran1"

    ReDim arr(3) As String  ' (3) means (0 To 3)

    arr(0)="0"  ' offset

    arr(1)="1"  ' ampl

    arr(2)="2"  ' freq

    arr(3)="3"  ' tdelay

    .SetPortSignal("1", "Sine", arr)  ' Pre-defined string array as argument

 

    ReDim arr(1 To 4) As String

    arr(1)="1"

    arr(2)="2"

    arr(3)="3"

    arr(4)="4"

    .SetPortSignal("1", "Sine", arr)  ' Pre-defined string array as argument (only size matters)

 

    ' 2. example for other signal type with different string array size

    .Reset

    .Name "Tran2"

    ReDim arr(3) As String  ' (3) means (0 To 3)

    arr(0)="0"  ' Fmin

    arr(1)="1"  ' Fmax

    arr(2)="2"  ' Ampl

    arr(3)="false"  ' Use FMin/Fmax from Task (optional)

    .SetPortSignal("1", "Gaussian", arr)  ' Need 3 definitions (optional 4) To define a gaussian signal

 

    ' 3. example to define a "Digital-Generic-PRBS random" signal

    .Reset

    .Name "Tran3"

    Dim values() As String

    ReDim values(9) As String  ' (9) means (0 To 9)

    values(0) = "Generic"      ' Shape Type

    values(1) = "PRBS random"  ' Pattern Type

    values(2) = "0"  ' Alow

    values(3) = "1"  ' Ahigh

    values(4) = "1"  ' Trise

    values(5) = "2"  ' Tfall

    values(6) = "3"  ' Tdelayrise

    values(7) = "4"  ' Tdelayfall

    values(8) = "13" ' Ttotal

    values(9) = "7"  ' N

    .SetPortSignal("1", "Digital", values)

    .SetPortSignalDelay("1", "1.0")

    .SetPortSourceType("1", "Signal")

    .SetPortSignalName("1", "Digital_1")

 

    ' 3. example to define a "Digital-Duty cycle and frequency-Manual" signal

    .Reset

    .Name "Tran4"

    Dim values() As String

    ReDim values(10) As String

    values(0) = "Duty cycle and frequency" ' Shape Type

    values(1) = "Manual"  ' Pattern Type

    values(2) = "0.0" ' Alow

    values(3) = "1.0" ' Ahigh

    values(4) = "0.1" ' Trise

    values(5) = "0.1 ' Tfall

    values(6) = "1" ' Duty cycle

    values(7) = "1" ' Freq

    values(8) = "0" ' Initbit

    values(9) = "101010" ' Bitsequence

    values(10) = "true" ' Periodic

    .SetPortSignal("1", "Digital", values)

    .SetPortSignalDelay("1", "1.0")

    .SetPortSourceType("1", "Signal")

    .SetPortSignalName("1", "Digital_1")

 

End With
